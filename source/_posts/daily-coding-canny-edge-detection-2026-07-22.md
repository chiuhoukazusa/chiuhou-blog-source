## 一、背景与动机

边缘检测是计算机视觉中最基础也最重要的预处理步骤之一。在图像理解、目标识别、医学影像分析等场景中，我们往往不需要完整的像素信息，只需要知道"物体的边界在哪里"。边缘本质上对应了图像亮度发生剧烈变化的位置——这些位置往往标志着物体的轮廓、纹理的边界或光照的跳变。

最朴素的边缘检测思路是计算梯度：用差分近似导数，梯度幅值大的地方就是边缘。1968 年 Sobel 算子问世，用一个 3×3 卷积核在 x 和 y 两个方向上计算梯度，成为了此后几十年间最常用的边缘检测方法。Sobel 算法简单高效，但它有几个致命的缺点：

1. **对噪声敏感**：没有预处理步骤，噪点也会产生高梯度，导致大量假边缘
2. **边缘过粗**：直接对梯度幅值取阈值会产生很宽的"带状"边缘，而不是锐利的单像素线
3. **阈值选择脆弱**：一个全局阈值无法同时适应图像中的强边缘和弱边缘——高了会漏掉弱边缘，低了会引入大量噪声

1986 年 John F. Canny 在他的硕士论文中提出了一个里程碑式的边缘检测算法，一举解决了上述所有问题。Canny 算法首次将边缘检测从一个"直觉式"的操作升级为一个有严格数学定义的优化问题。Canny 为"好的边缘检测"设定了三条准则：

- **好的检测率（Good Detection）**：真边缘不该被漏掉，假边缘不该被引入
- **好的定位精度（Good Localization）**：检测到的边缘位置应尽可能接近真实边缘
- **单一响应（Single Response）**：对于同一条边缘，检测器只应给出一个响应，不应该产生多个分离的像素点

这三条准则被 Canny 形式化为数学表达式，通过变分法推导出了最优的边缘检测算子，其结果近似为一个高斯函数的一阶导数——这正是 Canny 算法的理论基础。

在工业界，Canny 边缘检测被广泛使用：
- OpenCV 的 `cv::Canny()` 函数就是这个算法的实现
- 自动驾驶中用于车道线检测
- 医学影像中用于器官和肿瘤边界提取
- 工业视觉中用于产品缺陷检测

今天，我们将从零实现一个完整的 Canny 边缘检测器，不依赖任何第三方图像处理库——只使用 C++ 标准库，自己编写 PPM 图像 I/O 和所有算法步骤。

## 二、核心原理

Canny 边缘检测由五个顺序执行的步骤组成。每一步都基于上一步的输出，缺一不可。下面逐拆解每个步骤的数学原理。

### 2.1 高斯平滑（Gaussian Smoothing）

在计算梯度之前，必须抑制噪声。如果不对图像做平滑处理，噪点产生的高梯度会被误认为是边缘。

高斯滤波器是 Canny 的选择，因为它在空域和频域都有良好的局部化特性——不会像均值滤波器那样引入振铃伪影，也不会像中值滤波器那样可能破坏边缘的连续性。

二维高斯核的数学表达式为：

```
G(x, y) = (1 / (2πσ²)) · exp(-(x² + y²) / (2σ²))
```

其中 σ 控制平滑的程度。σ 越大，平滑越多，细节损失越多；σ 越小，保留细节越多，但降噪效果越差。Canny 建议 σ = 1.4，这是一个经过大量实验验证的折中值。

**直觉解释**：高斯核本质上是一个"加权平均"操作。它给中心像素最大的权重，给周围像素递减的权重。离中心越远的像素，对平滑结果的贡献越小。这种权重衰减服从正态分布——这就是"高斯"这个名字的由来。

在实际实现中，我们不需要在每次计算时都重新计算这个指数函数。只需要在初始化时生成一个 N×N 的核（通常是 5×5），然后对每个像素进行卷积即可。核的大小选择也很关键：过小（3×3）平滑不足，过大（7×7 或更大）会模糊掉真实边缘。5×5 是 Canny 论文中使用的大小，兼顾了降噪效果和边缘保持。

核生成后必须归一化：将所有元素除以它们的总和，使得核中所有权重之和为 1。这保证了平滑操作不会改变图像的整体亮度。

**为什么不是其他滤波器？**
- **均值滤波器**：对边缘和噪声一视同仁，会模糊掉真实边缘
- **中值滤波器**：保边能力强，但不适合作为梯度的前置处理，因为它会引入非线性，破坏梯度的连续性
- **双边滤波器**：保边效果好，但计算代价大，且 Canny 后续步骤（NMS+双阈值）本身就能很好地区分边缘和非边缘

### 2.2 梯度计算（Gradient Computation）

对平滑后的图像，使用 Sobel 算子计算每个像素的梯度。

Sobel 算子使用两个 3×3 的卷积核：

```
Gx = [-1  0  +1]      Gy = [-1  -2  -1]
     [-2  0  +2]           [ 0   0   0]
     [-1  0  +1]           [+1  +2  +1]
```

Gx 检测垂直边缘（左右两列差值的加权和），Gy 检测水平边缘（上下两行的加权和）。中心列的权重为 0，体现了 Sobel 的差分本质。

从这两个值可以计算出：

```
magnitude = sqrt(Gx² + Gy²)    // 梯度幅值：边缘有多"强"
direction = atan2(Gy, Gx)      // 梯度方向：边缘法线的方向
```

**直觉解释**：
- `magnitude` 越大，说明该位置的亮度变化越剧烈，越可能是边缘
- `direction` 指向亮度增加最快的方向——在边缘处，这个方向垂直于边缘线

注意：梯度方向指向的是亮度增加的方向（从暗到亮），这与边缘方向相差 90 度——边缘线本身是沿着"等亮度线"的方向。但这对 NMS 步骤来说没有影响，因为我们只需要知道沿哪个方向比较像素。

### 2.3 非极大值抑制（Non-Maximum Suppression, NMS）

这是 Canny 算法中最关键的一步，也是它区别于简单梯度阈值法的地方。

原始的梯度幅值图中，边缘附近的多个像素都会有较高的幅值，形成一条"粗线"。人类看到的是"边缘"，但计算机看到的是"一片高梯度区域"。NMS 的目的就是把这片区域变成一条细细的线——只保留局部最大值。

**算法步骤**：

1. 将梯度方向量化到 4 个扇区（每个扇区 45 度）：0°、45°、90°、135°
2. 对于每个像素，沿着它的梯度方向（注意是梯度方向，不是边缘方向）比较相邻两个像素的幅值
3. 如果当前像素的幅值大于等于两个相邻像素的幅值，保留；否则置零

**方向量化细节**：

```
0°   (水平边缘):  方向 ∈ [0°, 22.5°) ∪ [157.5°, 180°]  → 比较 (x-1, y) 和 (x+1, y)
45°  (对角边缘):  方向 ∈ [22.5°, 67.5°)                 → 比较 (x+1, y-1) 和 (x-1, y+1)
90°  (垂直边缘):  方向 ∈ [67.5°, 112.5°)                → 比较 (x, y-1) 和 (x, y+1)
135° (反斜对角):  方向 ∈ [112.5°, 157.5°)               → 比较 (x-1, y-1) 和 (x+1, y+1)
```

**为什么量化到 4 个方向就够了？** 因为像素网格是离散的，我们最多只能精确地比较 8 个方向上的邻居。4 个扇区（8 个方向）已经覆盖了所有可能的情况。

**为什么用梯度方向而不是边缘方向？** 因为梯度的定义就是变化最快的方向。沿着这个方向的像素是"在同一座山坡上的相邻点"——我们比较它们的高度（梯度幅值），只留下山顶的像素。如果换成边缘方向（梯度垂直方向），那就成了沿着等坡度线比较，毫无意义。

NMS 之后，边缘线的宽度被压缩到 1-2 个像素，但问题还没解决：如何从剩余的候选像素中筛选出真正的边缘？

### 2.4 双阈值（Double Thresholding）

NMS 之后的图像中，有些像素确实对应真实边缘（幅值高），有些是噪声残留（幅值低）。简单的单阈值方法无法同时处理好两者。

Canny 创造性地提出了双阈值策略，灵感来自于人类视觉系统的对比敏感度感知：

- **高阈值（T_high）**：超过此阈值的像素 → **强边缘（Strong Edge）**，几乎确定是真实边缘
- **低阈值（T_low）**：在 T_low 和 T_high 之间的像素 → **弱边缘（Weak Edge）**，可能是边缘也可能是噪声
- 低于 T_low 的像素 → 直接丢弃

**阈值如何选择？**

Canny 的原始论文建议使用直方图分位数法：T_high 取梯度幅值直方图的 85% 分位数（即只有 15% 的梯度像素超过此值），T_low 取 T_high 的 40%。

在我们的实现中，使用了一个稍微宽松的参数：高阈值对应 85% 分位数，低阈值对应高阈值的 40%。

**为什么不用固定的绝对阈值？** 因为不同图像的对比度和亮度差异巨大。一张夜景照片中的边缘梯度幅值可能只有 30-50，而一张强光下的照片可能高达 200+。分位数方法自适应地根据图像内容调整阈值，鲁棒性更强。

### 2.5 滞后边缘追踪（Hysteresis Edge Tracking）

这是最后的也是最精巧的一步。双阈值之后，图像中有三种像素：255（强边缘）、128（弱边缘）、0（非边缘）。弱边缘像素的状态取决于它与强边缘的连通性。

**核心思想**：如果一条弱边缘与一条强边缘在空间上是连通（8 连通）的，那么它就是真实边缘的延续；如果一片弱边缘完全孤立，没有连接到任何强边缘，那它很可能只是噪声。

**实现方法——泛洪（Flood Fill / DFS）**：

1. 遍历所有像素，找到强边缘（值为 255 的像素）
2. 从每个强边缘像素出发，做 8 连通的深度优先搜索
3. 搜索过程中，遇到弱边缘（值为 128）就把它升级为强边缘（255）
4. 最终，所有仍为 128 的像素被归零（当作噪声丢弃）

**"滞后（Hysteresis）"这个名称的由来**：就像物理中的磁滞效应——物体被磁化后，即使磁场减弱也不会立刻消磁。在这里，"强边缘"像一块磁铁，吸引周围的"弱边缘"加入；但一旦某个弱边缘被判定为噪声，它就无法回头了。这种非对称的行为正是"滞后"的含义。


## 三、实现架构

### 3.1 整体数据流

```
输入图像 (PPM)
     │
     ▼
[Stage 1] 高斯平滑 ───→ stage1_gaussian.ppm（调试：验证平滑效果）
     ▼
[Stage 2] Sobel 梯度 ──→ stage2_gradient.pgm（保存梯度幅值用于可视化）
     ▼
[Stage 3] 非极大值抑制 ─→ stage3_nms.pgm（保存NMS后的图像用于对比）
     ▼
[Stage 4] 双阈值
     ▼
[Stage 5] 滞后边缘追踪
     ▼
输出图像 (PPM)
```

每一步的输出都单独保存为图像文件，方便调试和博客演示。

### 3.2 关键数据结构

```cpp
struct Image {
    int w, h;
    std::vector<unsigned char> r, g, b;  // 分离的三通道
};

struct GradientResult {
    std::vector<float> magnitude;   // 梯度幅值（浮点，保留精度）
    std::vector<float> direction;   // 梯度方向（弧度，[-π, π]）
    int w, h;
};
```

**设计理由**：
- `Image` 使用分离的 R/G/B 通道：与 PPM 格式读写对齐；后续做灰度处理时只需要计算加权和
- `GradientResult` 使用 `float` 而不是 `unsigned char`：梯度幅值可能超过 255，用 float 避免精度损失
- `magnitude` 和 `direction` 使用独立数组而不是 `std::pair`：内存布局更紧凑，缓存友好

### 3.3 图像 I/O 设计

项目使用 PPM（Portable Pixmap）格式。选择 PPM 的原因：
- 纯文本头 + 二进制数据，读写极简（~30 行代码）
- 无损存储，不会有 JPEG 压缩伪影干扰边缘检测
- 不需要任何外部库

P6 格式的 PPM 文件结构：
```
P6
512 512        ← 宽度 高度
255            ← 最大值
<512×512×3 字节的 RGB 数据>
```

### 3.4 职责划分

整个流程全部在 CPU 端完成。每个 stage 都是纯函数：输入一张图，输出一张图，不修改输入。

- **PPM 读写**：`readPPM()`, `writePPM()`, `writePGM()` —— 仅负责文件格式解析
- **Stage 函数**：`gaussianBlur()`, `computeGradients()`, `nonMaximumSuppression()`, `doubleThreshold()`, `hysteresisEdgeTracking()` —— 纯算法逻辑
- **验证函数**：`computeEdgeStats()` —— 量化验证，不依赖人工视觉检查
- **测试图案**：`generateTestImage()` —— 包含矩形、圆形、菱形、细线、渐变区域

## 四、关键代码解析

### 4.1 高斯核生成

```cpp
std::vector<float> createGaussianKernel(int size, float sigma) {
    std::vector<float> kernel(size * size);
    int half = size / 2;
    float sum = 0.0f;
    
    for (int y = -half; y <= half; y++) {
        for (int x = -half; x <= half; x++) {
            float val = expf(-(x*x + y*y) / (2.0f * sigma * sigma));
            kernel[(y+half)*size + (x+half)] = val;
            sum += val;
        }
    }
    
    for (int i = 0; i < size * size; i++)
        kernel[i] /= sum;
    
    return kernel;
}
```

**为什么这样写**：
- 使用 `expf()`（单精度指数）而不是 `exp()`：在 5×5 核上的精度足够，且速度更快
- `x*x + y*y` 比 `pow(x, 2) + pow(y, 2)` 快得多：编译器对整数的平方有优化
- 归一化放在最后：避免每个元素除以 running sum（会导致累积误差）
- 二维核展开为一维数组：缓存局部性最好

**容易踩的坑**：高斯核生成时，如果忘记归一化（除以 sum），输出的图像会整体变亮或变暗。

### 4.2 高斯卷积的核心循环

```cpp
for (int y = 0; y < src.h; y++) {
    for (int x = 0; x < src.w; x++) {
        float sum = 0;
        for (int ky = -half; ky <= half; ky++) {
            for (int kx = -half; kx <= half; kx++) {
                int px = std::clamp(x + kx, 0, src.w - 1);
                int py = std::clamp(y + ky, 0, src.h - 1);
                float gray = src.r[py*src.w + px] * 0.299f + 
                             src.g[py*src.w + px] * 0.587f + 
                             src.b[py*src.w + px] * 0.114f;
                sum += gray * kernel[(ky+half)*kernelSize + (kx+half)];
            }
        }
        // ...
    }
}
```

**为什么这样写**：
- `std::clamp(..., 0, src.w-1)` 处理边界：当核延伸到图像外部时，clamp 到最近的边界像素。另一种方案是跳过边界，但这会在边缘留下未处理的条带
- `0.299f*R + 0.587f*G + 0.114f*B` 是 ITU-R BT.601 标准的人眼亮度感知加权系数。绿色系数最大（0.587），因为人眼对绿色光最敏感
- 每个像素都要重新读取其 RGB 值计算灰度，而不是先做一次完整的 RGB→灰度转换：因为我们只需要为平滑后的结果生成灰度图，两步合成一步减少一次全图遍历

### 4.3 Sobel 梯度计算

```cpp
const int Gx[3][3] = {{-1, 0, 1}, {-2, 0, 2}, {-1, 0, 1}};
const int Gy[3][3] = {{-1,-2,-1}, { 0, 0, 0}, { 1, 2, 1}};

for (int y = 1; y < src.h - 1; y++) {
    for (int x = 1; x < src.w - 1; x++) {
        float sx = 0, sy = 0;
        for (int ky = -1; ky <= 1; ky++) {
            for (int kx = -1; kx <= 1; kx++) {
                unsigned char val = src.r[(y+ky)*src.w + (x+kx)];
                sx += val * Gx[ky+1][kx+1];
                sy += val * Gy[ky+1][kx+1];
            }
        }
        gr.magnitude[y*src.w + x] = sqrtf(sx*sx + sy*sy);
        gr.direction[y*src.w + x] = atan2f(sy, sx);
    }
}
```

**为什么这样写**：
- 使用 `atan2f(sy, sx)` 而不是 `atan(sy/sx)`：`atan2f` 能正确处理 `sx=0` 的情况（不会除零），并且返回值范围是 `[-π, π]`（覆盖全象限），而 `atan` 只能返回 `[-π/2, π/2]`
- 循环从 `(1, 1)` 到 `(w-2, h-2)`：跳过边界像素，因为 Sobel 3×3 核需要访问所有邻居
- `sx` 和 `sy` 使用了 `int` 类型核乘法：Sobel 核的值是固定整数，不需要用浮点核

**容易疏漏的地方**：如果把 Gx 和 Gy 写反了（即用 Gy 的核去算 Gx），输出不会报错，但梯度方向会偏差 90 度——这会导致 NMS 在错误的方向上做抑制，产生断裂的边缘线。

### 4.4 非极大值抑制——方向量化

```cpp
float angle = gr.direction[y*gr.w + x] * 180.0f / M_PI;
if (angle < 0) angle += 180.0f;  // 标准化到 [0, 180]

// Sector 0 (0° gradient direction → horizontal comparison)
if ((angle >= 0 && angle < 22.5) || angle >= 157.5) {
    n1 = gr.magnitude[y*gr.w + (x-1)];
    n2 = gr.magnitude[y*gr.w + (x+1)];
}
// Sector 1 (45° diagonal)
else if (angle >= 22.5 && angle < 67.5) {
    n1 = gr.magnitude[(y-1)*gr.w + (x+1)];
    n2 = gr.magnitude[(y+1)*gr.w + (x-1)];
}
// Sector 2 (90° gradient → vertical comparison)
else if (angle >= 67.5 && angle < 112.5) {
    n1 = gr.magnitude[(y-1)*gr.w + x];
    n2 = gr.magnitude[(y+1)*gr.w + x];
}
// Sector 3 (135° anti-diagonal)
else {
    n1 = gr.magnitude[(y-1)*gr.w + (x-1)];
    n2 = gr.magnitude[(y+1)*gr.w + (x+1)];
}

unsigned char val = (mag >= n1 && mag >= n2) 
    ? (unsigned char)std::min(mag, 255.0f) : 0;
```

**为什么这样写**：
- 方向角先归一化到 `[0°, 180°)`：因为边缘的"朝向"在 0°→180° 和 180°→360° 是等价的
- 扇区边界 22.5°/67.5°/112.5°/157.5°：刚好是 0°/45°/90°/135° 的 ±22.5° 范围。22.5° 是 45° 的一半
- `>=` 比较包括等号：如果当前像素的幅值等于邻居的幅值，我们保留它

### 4.5 双阈值——分位数法

```cpp
std::vector<float> mags;
for (int i = 0; i < nms.w * nms.h; i++) {
    if (nms.r[i] > 0) mags.push_back((float)nms.r[i]);  // 过滤零值！
}

std::sort(mags.begin(), mags.end());
float highThresh = mags[(int)(mags.size() * (1.0f - highRatio))];
float lowThresh = highThresh * lowRatio;
```

**为什么用分位数而不是固定值**：
- 不同场景下梯度的动态范围差异巨大。分位数方法自适应地根据图像内容调整阈值
- `highRatio=0.15` 表示取 85% 分位数：只有前 15% 最强的梯度被认为是强边缘
- `lowRatio=0.4`：低阈值是高阈值的 40%。太高会丢失弱边缘，太低会引入噪声

**容易踩的坑**：NMS 之后大多数像素的幅值都是 0。如果包含这些 0，分位数会被拉低到接近 0。所以代码中先 `if (nms.r[i] > 0)` 过滤了这些零值——这是实际编码中踩过的最严重的坑。

### 4.6 滞后边缘追踪——DFS 泛洪

```cpp
void edgeTrackDFS(Image& img, int x, int y) {
    const int dx[8] = {-1, 0, 1, -1, 1, -1, 0, 1};
    const int dy[8] = {-1, -1, -1, 0, 0, 1, 1, 1};
    
    img.r[y*img.w + x] = 255;  // 升级为强边缘
    img.g[y*img.w + x] = 255;
    
    for (int i = 0; i < 8; i++) {
        int nx = x + dx[i];
        int ny = y + dy[i];
        if (nx >= 0 && nx < img.w && ny >= 0 && ny < img.h) {
            if (img.r[ny*img.w + nx] == 128) {
                edgeTrackDFS(img, nx, ny);
            }
        }
    }
}
```

**为什么用 DFS 而不是 BFS**：
- DFS 的栈空间在 512×512 尺度下完全可以接受
- DFS 的访问顺序是"一条道走到黑"，这恰好符合边缘追踪的直觉
- 如果害怕爆栈，可以改用显式栈或 BFS

**为什么 8 连通而不是 4 连通**：4 连通只检查上下左右，会丢失对角方向的边缘连续性。一个 45° 倾斜的边缘线，在 4 连通下会变成断裂的虚线。

**DFS 被触发的条件**：主循环中，只有在当前像素是强边缘（255）时，才检查它的 8 个邻居中是否有弱边缘（128）。如果有，就调用 DFS 去"征服"它们。

## 五、踩坑实录

### Bug 1：NMS 方向量化错误——边缘断裂

**症状**：NMS 输出的边缘出现明显的断裂，特别是在 45° 方向的边缘上。每隔几个像素就有一个断点。整体边缘密度只有预期的 30%。

**错误假设**：最初我以为梯度方向和边缘方向是同一个概念，所以在比较邻居时用的是"边缘方向"（与梯度方向垂直）。也就是说，当梯度方向是 90° 时，我比较的是 `(x-1, y)` 和 `(x+1, y)`（即水平方向）。

**真实原因**：NMS 需要沿着梯度方向比较——梯度方向是"亮度变化最快的方向"，也是"边缘法线的方向"。沿着这个方向，像素的梯度幅值呈现出一个"山峰"的形状，我们要保留"山顶"的像素。如果沿着边缘方向比（即水平地沿着山峰的脊线比），所有像素的幅值几乎相同，NMS 不会有任何效果。

**修复**：把比较的方向改为梯度方向，参考对应的扇区映射表。修复后边缘恢复了连续性。

### Bug 2：分位数计算包含了零值——阈值过低

**症状**：双阈值计算出的高阈值异常低（< 10），导致 90% 以上的 NMS 输出像素被标记为强边缘。最终边缘密度高达 45%，图像几乎全白。

**错误假设**：我直接用 `std::sort` 对所有 NMS 输出像素（包括值为 0 的）排序，然后取 85% 分位数。我以为 NMS 之后大多数像素应该是零，取 85% 分位数自然是一个很高的值。

**真实原因**：NMS 之后，被抑制的像素值为 0。0 值占图像的绝大多数（在我的测试图中，NMS 后只有约 1.6% 的像素非零，98.4% 为零）。如果所有零值都参与排序，那么 85% 分位数实际指向的是第 85 个元素——假设 100 个像素，其中 98 个是 0，2 个是 200。按升序排列后，第 85 个是 0。取 85% 分位数得到的还是 0！这就意味着任何高于 0 的梯度幅值都会被判定为强边缘——简直就是灾难。

**修复**：在计算分位数之前，先过滤掉值为 0 的元素。`if (nms.r[i] > 0) mags.push_back(...)` 这一行是关键。修复后，分位数只考虑"有希望成为边缘"的像素，阈值恢复正常。

### Bug 3：Hysteresis 只在第一次遍历时 Trace

**症状**：滞后追踪后，从强边缘延伸出的弱边缘线只追踪了第一层（8 个直接邻居）。更深层的弱边缘没有被追踪，导致中等长度的弱边缘线在 2-3 个像素后就断裂了。

**错误假设**：第一版代码在主循环中只检查了强边缘像素的 8 个直接邻居，遇到弱边缘就把它设为强边缘。我以为这样设置之后，在下一次遍历中这个新升级的强边缘会自动触发新一轮追踪。

**真实原因**：弱边缘被升级为强边缘后，它的邻居在同样的遍历中可能已经被处理过了（因为遍历是从上到下、从左到右的）。例如，一个强边缘在 (x, y)，它的弱边缘邻居在 (x+1, y) 被升级。但 (x+1, y+1) 的弱边缘需要一个"新强边缘"在 (x, y+1) 被处理时才能被发现——而 (x, y+1) 可能已经被遍历过了。

**修复**：改用 DFS 递归。每当发现一个弱边缘连接到强边缘时，立即递归地追踪这个弱边缘的所有 8 个邻居。递归的终止条件是：所有邻居要么不是弱边缘，要么超出边界。

## 七、总结与延伸

### 7.1 Canny 算法的局限性

Canny 虽然优秀，但并非完美：

1. **对参数敏感**：σ、核大小、高/低阈值的选择直接影响结果。OpenCV 版本提供了自动阈值计算的 `cv::Canny()` 重载，但有时仍需要手动调参。

2. **不适用的场景**：
   - **纹理密集的图像**（如草地、织物）：会产生大量密集边缘，结果难以解读
   - **渐变区域**：Canny 对渐变理论上不应有响应，但在噪声存在的情况下，渐变区域局部梯度可能超过阈值
   - **低光照图像**：噪声水平接近信号水平，信噪比低时任何边缘检测方法都难以为继

3. **计算开销**：完整的 5 步流程比单步骤的 Sobel 阈值法慢很多。5×5 高斯卷积 + Sobel + NMS + 双阈值 + DFS，每个步骤都需要遍历全图。

### 7.2 可优化方向

- **可分离卷积**：将 5×5 高斯核分解为两个 1×5 和 5×1 的一维卷积，复杂度从 O(N×K²) 降到 O(2N×K)
- **SIMD 加速**：利用 SSE/AVX 指令集并行处理多个像素的卷积运算
- **自适应阈值**：根据图像局部的对比度动态调整双阈值（类似 CLAHE 的思路）
- **GPU 并行**：将每个 stage 实现为 Compute Shader，在 GPU 上并行执行

### 7.3 与系列其他文章的关联

Canny 边缘检测是图像处理系列的第三篇，前两篇分别是：
- **07-20 Sobel Edge Detection**：实现了最基础的梯度边缘检测，但没有 NMS 和滞后追踪
- **07-21 Median Filter Denoising**：实现了中值滤波降噪，可以与高斯平滑做对比

Canny 算法相当于把 Sobel 的梯度计算、高斯平滑的降噪、非极大值抑制的细化、双阈值的筛选、滞后追踪的连通性分析全部整合在一起——它是边缘检测领域的集大成者。
