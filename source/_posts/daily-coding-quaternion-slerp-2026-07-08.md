## 背景与动机

如果你做过任何 3D 开发，一定遇到过「旋转」这件事。看似简单——一个物体绕某个轴旋转就行了，对吧？但实际上这是图形学中最容易踩坑的领域之一。

**欧拉角的困境。** 传统上，我们用三个角度来表示旋转：绕 X 轴转多少度（pitch）、绕 Y 轴转多少度（yaw）、绕 Z 轴转多少度（roll）。这就是欧拉角表示法。这种方法直观易懂，Unity 编辑器的 Transform 组件用的就是它。但在实际开发中，欧拉角有一个致命的缺陷：**万向节死锁（Gimbal Lock）**。

**什么是万向节死锁？** 想象你在操作一个摄像机。正常情况下，三个轴的旋转应该是独立的。但当 pitch（绕 X 轴旋转）恰好等于 90° 时，原来的 Y 轴（yaw）和 Z 轴（roll）会「塌缩」到同一根轴上——你失去了一个旋转自由度。从这一刻起，无论你动 yaw 还是 roll，物体都在绕同一根轴旋转。这对于飞行模拟器、3D 动画、机器人运动学来说是灾难性的——你无法在 pitch=90° 时表达某些朝向。

在游戏引擎中，这个问题无处不在：
- **摄像机控制**：第一人称视角中，如果垂直向上看（pitch=90°），水平转动会变得异常
- **骨骼动画**：角色关节的旋转插值如果用欧拉角，在关节折叠到极限角度时会出现奇怪的扭曲
- **飞行模拟**：战斗机做垂直爬升时，如果导航系统用欧拉角，会出现数值不稳定的情况

**四元数的登场。** 1843 年，爱尔兰数学家 William Rowan Hamilton 在都柏林的布鲁姆桥上灵光一闪，在石头上刻下了四元数的基本公式：`i² = j² = k² = ijk = -1`。这个几乎被遗忘的数学工具，在 20 世纪末被计算机图形学界重新发现——它天然规避了万向节死锁，而且旋转插值（SLERP）极其优雅。

**为什么不用旋转矩阵直接插值？** 旋转矩阵是 3×3 的 9 个数字，直接对矩阵元素做线性插值得到的结果通常不是合法的旋转矩阵（不满足正交性和行列式为 1），会产生缩放和剪切畸变。四元数把旋转压缩成 4 个数，在 4D 单位球面上做球面插值，结果是完美合法的旋转。

今天这篇文章，我们从零推导四元数的旋转机制，实现 SLERP 球面线性插值，并且用定量化的方式验证结果的正确性——不只是画几个漂亮的图然后说「看起来还行」。

## 核心原理

### 四元数的数学定义

一个四元数 `q` 定义为：

```
q = w + xi + yj + zk
```

其中 `w, x, y, z` 是实数，`i, j, k` 是三个虚数单位，满足以下乘法规则：

```
i² = j² = k² = ijk = -1
```

从这三个基本等式可以推导出：

```
ij = k,   ji = -k
jk = i,   kj = -i
ki = j,   ik = -j
```

**关键直觉**：四元数乘法不满足交换律！`ij = k` 但 `ji = -k`。这种「反交换」性质正是四元数能表示 3D 旋转的关键——它天然编码了旋转的顺序依赖。

### 为什么用 4 个数表示 3D 旋转？

这个问题值得仔细解释。3D 旋转只有 3 个自由度（绕 X、Y、Z 轴的转角），为什么我们需要 4 个数的四元数？

**答案是：3 个数不够用，会导致奇点。** 欧拉角之所以出现万向节死锁，本质是 3 参数的参数化在覆盖 SO(3)（三维旋转群）时存在拓扑上的障碍——通俗地说，就是 SO(3) 这个「空间」不能用 3 个连续的全局坐标来完美描述。这就像你不能用一张完整的地图来覆盖整个地球表面而不出现重叠或拉伸——总会有极点附近出现变形（墨卡托投影在南北极无限拉伸）。欧拉角在 pitch=±90° 时出现的死锁，就是这种拓扑障碍的体现——就像北极点附近经度的定义变得模糊。

四元数用 4 个数加上「归一化」这个约束（`w²+x²+y²+z²=1`），恰好提供了所需的 3 个自由度。多出来的这一维不是浪费，而是给了我们数学上的「冗余度」来避免奇点。类比来讲：绕任意轴旋转可以用轴（2 个自由度，单位方向向量）加上旋转角（1 个自由度），3 个自由度。四元数把轴和角编码在一起，提供了一个紧凑且无奇点的表示。

### 轴角表示到四元数的转换

从几何直觉出发：**任何 3D 旋转都可以描述为绕某个单位轴 `u=(ux,uy,uz)` 旋转角度 `θ`。**

对应的四元数构造为：

```
q = cos(θ/2) + sin(θ/2)·(ux·i + uy·j + uz·k)
```

写成代码形式：

```
q.w = cos(θ / 2)
q.x = ux · sin(θ / 2)
q.y = uy · sin(θ / 2)
q.z = uz · sin(θ / 2)
```

**为什么是 θ/2？** 这是四元数旋转中最不直观的一点，也是新手最容易困惑的地方。直觉解释：四元数旋转一个向量时，我们使用「两次乘法」：`v' = q·v·q⁻¹`。这里的 `q` 本质上是在 4D 空间中做了一个「双重反射」，每层反射贡献 θ/2 的旋转，两层合起来正好是 θ。如果你直接拿 θ 来构造四元数，旋转就会是 2θ。

**验证方式**：考虑绕 Z 轴旋转 180°（θ=π），轴 u=(0,0,1)。代入公式：
```
q = cos(π/2) + sin(π/2)·k = 0 + 1·k = k
```
用 q=k 旋转向量 v=(x,y,z)，公式 `q·v·q⁻¹` = `k·(xi+yj+zk)·(-k)` = ... = -xi - yj + zk。X 和 Y 分量翻转，Z 不变——确实是绕 Z 轴 180°。验证通过。

### 四元数乘法（Hamilton Product）

两个四元数 `q1 = (w1, x1, y1, z1)` 和 `q2 = (w2, x2, y2, z2)` 的乘积：

```
q1 × q2 = (
    w1·w2 - x1·x2 - y1·y2 - z1·z2,
    w1·x2 + x1·w2 + y1·z2 - z1·y2,
    w1·y2 - x1·z2 + y1·w2 + z1·x2,
    w1·z2 + x1·y2 - y1·x2 + z1·w2
)
```

**记忆力技巧**：把这个公式分成两部分看：
- **w 分量**：点积减点积——`w1·w2 - (x1·x2 + y1·y2 + z1·z2)`，也就是标量部分做点积，向量部分做点积取负
- **x, y, z 分量**：标量乘向量 + 标量乘向量 + 叉积。具体来说，`w1` 乘对方向量分量 + `w2` 乘自己的向量分量 + 自己向量与对方向量的叉积

掌握这个结构后不需要死记硬背。

**为什么四元数乘法能表示旋转的组合？** 如果你先做旋转 q1，再做旋转 q2，等效变换是 `q2·(q1·v·q1⁻¹)·q2⁻¹`。利用乘法结合律，这等价于 `(q2·q1)·v·(q2·q1)⁻¹`。所以两个旋转的**组合**就是四元数**乘法**（注意顺序——后应用的写在前面）。

### 旋转向量

用法：`v' = q·v·q⁻¹`，其中 `v` 作为纯四元数 `(0, vx, vy, vz)`。

这一步的直觉：把四元数看成在 4D 空间中做了某种「旋转」，这个旋转投影回 3D 就得到了我们想要的 3D 旋转。具体来说，四元数旋转实际上是两次 4D 反射的组合（Cartan-Dieudonné 定理的体现）。

### SLERP：球面线性插值

这是四元数真正闪光的地方。

**问题**：如何平滑地从旋转 A 过渡到旋转 B？

如果我们用欧拉角：对三个角度分别线性插值 → 中间旋转可能不自然，甚至出现万向节死锁。

**SLERP 的原理**：把两个单位四元数 `qa` 和 `qb` 看作 4D 单位球面上的两个点。SLERP 沿球面的大圆（短弧）以恒定角速度插值：

```
SLERP(qa, qb, t) = (sin((1-t)·Ω)·qa + sin(t·Ω)·qb) / sin(Ω)
```

其中 `Ω = arccos(qa·qb)` 是两个四元数之间的角度（通过点积计算）。

**直觉**：这相当于在 4D 球面上以恒定角速度从 qa 走到 qb。插值结果始终在单位球面上，因此始终是合法的旋转。不会出现旋转矩阵直接插值产生的「缩放」和「剪切」。

**最短路径处理**：四元数有一个特性：q 和 -q 表示相同的旋转（因为 `(-q)·v·(-q)⁻¹ = q·v·q⁻¹`）。这意味着在 4D 球面上，qa 到 qb 有两条路径——大圆的短弧和长弧。我们选择短弧：如果 `qa·qb < 0`，就把 qb 取反（`qb = -qb`），这样角度 `< π/2`，走短路径。

**当角度非常小时的退化处理**：当 `qa ≈ qb` 时，`sin(Ω) ≈ 0`，除法会爆炸。此时退化为普通线性插值（NLERP）：

```
nlerp(qa, qb, t) = normalize(qa + t·(qb - qa))
```

这里的融合方法是：在小角度（`cosTheta > 1-ε`）时用 NLERP 替代 SLERP，差值微乎其微。

### SLERP vs NLERP

NLERP（Normalized Linear Interpolation）是先把四元数做线性插值再归一化。它在大部分情况下看起来还行，但有个问题——**角速度不恒定**。在起点和终点附近旋转得快，在中间旋转得慢。对于动画和摄像机控制来说，这种不均匀的旋转会让运动看起来「卡顿」或「不线性」。SLERP 保证恒定角速度，视觉效果更流畅。

### 四元数转旋转矩阵

对于需要和传统渲染管线集成的场景，需要将四元数转换为 3×3 的旋转矩阵：

```
R = [
    1-2(y²+z²)     2(xy-wz)      2(xz+wy)   ]
    2(xy+wz)       1-2(x²+z²)    2(yz-wx)   ]
    2(xz-wy)       2(yz+wx)      1-2(x²+y²) ]
```

**验证性质**：旋转矩阵必须是正交矩阵且行列式为 1：
- `R × R^T = I`（正交性检查）
- `det(R) = 1`（是纯旋转，不含反射）

这两个条件是我们在代码中做的定量验证。

## 实现架构

### 整体数据流

```
欧拉角/轴角 → 四元数构造 → 归一化 → 旋转操作/插值(SLERP) → 旋转矩阵转换 → 3D顶点变换 → 2D投影 → PPM渲染
```

整个管线可以分为三个层次：

1. **表示层（Representation）**：Quat 结构体 + Vec3 结构体
2. **操作层（Operation）**：乘法、插值（SLERP/NLERP）、旋转向量
3. **验证层（Verification）**：范数检查、round-trip 测试、正交性测试、角速度一致性测试

### 关键数据结构设计

**Vec3**：标准的 3D 向量，提供了向量空间操作（加减、叉积、点积、归一化），这里不再赘述。

**Quat**：四元数结构体，4 个双精度浮点 `(w, x, y, z)`：
- 默认构造函数初始化为恒等四元数 `(1, 0, 0, 0)`——表示「不旋转」
- `fromAxisAngle(axis, angle)`：从任意轴角构造四元数
- `fromEuler(yaw, pitch, roll)`：从欧拉角构造（ZYX 顺序），用于与欧拉角对比
- `normalized()`：返回归一化的四元数。在多次乘法后浮点误差会积累，导致范数偏离 1，因此需要在 SLERP 中间步骤做归一化
- `conjugate()`：共轭，用于旋转向量的第二步
- `rotate(v)`：将 3D 向量 v 用此四元数表示的方向旋转
- `toMatrix(m)`：转换为 3×3 旋转矩阵

### 渲染管线

采用正交投影的简化管线：
1. 从 cube 的 8 个顶点出发
2. 用 Quat::rotate() 旋转每个顶点
3. 加上偏移量（放在画布的特定位置）
4. 正交投影到 2D：`(x, y) → (W/2 + x·scale·W, H/2 - y·scale·H)`
   - 注意 Y 轴取反：世界坐标 Y 向上，屏幕坐标 Y 向下
5. 用 Bresenham 画线算法绘制 12 条边

之所以不写完整的 3D 渲染管线（透视投影、深度测试），是因为本篇的核心是展示四元数的旋转特性，不是一个三角形光栅化教程。

### 六个验证区段

代码的 main 函数分为六个区段，每个都输出定量数据：

| 区段 | 内容 | 验证方式 |
|------|------|---------|
| 1 | SLERP 插值可视化 | 输出每个插值点的旋转角度 |
| 2 | NLERP vs SLERP 对比 | 输出两者间的角度差 |
| 3 | 万向节死锁演示 | 对比欧拉角 LERP 和四元数 SLERP |
| 4 | Axis-Angle ↔ Quat Round-trip | 42 个角度×轴组合的范数+往返测试 |
| 5 | 旋转矩阵正交性检查 | R×R^T - I 的最大误差 + det(R) |
| 6 | SLERP 角速度一致性 | 逐段角度的标准差检查 |

## 关键代码解析

### 1. 轴角到四元数

```cpp
static Quat fromAxisAngle(const Vec3& axis, double angle) {
    double half = angle * 0.5;
    double s = std::sin(half);
    Vec3 a = axis.normalized();
    return Quat(std::cos(half), a.x * s, a.y * s, a.z * s);
}
```

**关键点**：角度必须先除以 2（`half = angle * 0.5`），原因如前所述——四元数的「双重覆盖」性质。如果你忘了除以 2，旋转角度会变成 2 倍∠。

**为什么先归一化 axis？** 如果传入的轴向量不是单位长度（比如 `(1,1,0)` 长度是 √2），sin(half)·非单位向量会导致四元数范数不为 1——这会让后续的旋转操作产生缩放效果，违反「单位四元数表示纯旋转」的前提。所以必须先归一化，这也是最容易出错的地方之一。

### 2. SLERP 核心实现

```cpp
Quat slerp(const Quat& a, const Quat& b, double t) {
    t = std::max(0.0, std::min(1.0, t));  // 安全限幅
    
    double cosTheta = a.dot(b);
    
    // 最短路径：如果余弦为负，反向其中一个
    Quat qb = b;
    if (cosTheta < 0) {
        qb = Quat(-b.w, -b.x, -b.y, -b.z);
        cosTheta = -cosTheta;
    }
    
    // 角度太小则退化到 NLERP，避免除以零
    if (cosTheta > 1.0 - EPS) {
        return Quat(
            a.w + t*(qb.w - a.w),
            a.x + t*(qb.x - a.x),
            a.y + t*(qb.y - a.y),
            a.z + t*(qb.z - a.z)
        ).normalized();
    }
    
    double theta = std::acos(cosTheta);
    double sinTheta = std::sin(theta);
    
    double s0 = std::sin((1 - t) * theta) / sinTheta;
    double s1 = std::sin(t * theta) / sinTheta;
    
    return Quat(
        s0*a.w + s1*qb.w,
        s0*a.x + s1*qb.x,
        s0*a.y + s1*qb.y,
        s0*a.z + s1*qb.z
    );
}
```

逐步骤讲解：

**第一步：`t` 限幅。** 虽然 SLERP 在数学上定义在 `t ∈ [0,1]`，但在动画系统中 t 可能因为数值精度略微超出范围。在边界外做 `acos` 会返回 NaN——这是常见的崩溃来源。

**第二步：计算余弦。** `a.dot(b)` 直接衡量两个四元数在 4D 球面上的夹角。值为 1 时表示相同旋转，值为 -1 时表示相反旋转（实际表示相同的朝向，但走另一条路径），值为 0 时表示正交（夹角 90°）。

**第三步：最短路径选择。** 这是实现中**最容易遗漏**的细节。如果你不检查 `cosTheta < 0` 并取反，SLERP 就会走球面上的「远程路线」——旋转角度将大于 180°，物体看起来会走一条怪异的大圈而不是最短路径。这在动画中会导致角色关节突然从关节角度 A 翻转 270° 而不是平滑地转动 90°。

**第四步：小角度退化。** 当两个四元数几乎相同时（比如关键帧之间变化很小），`theta ≈ 0`，`sin(theta) ≈ 0`，除法产生巨大数值。此时用 NLERP 替代 SLERP，结果几乎没有可见差异。阈值 `EPS = 1e-9` 对应夹角约 `8e-5` 度——远低于任何可见精度。

**第五步：球面插值。** 标准 SLERP 公式。`s0` 和 `s1` 是球面上的重心坐标（在球面几何中的广义插值权重），它们满足 `s0² + 2·s0·s1·cos(θ) + s1² = 1`（归一化条件）。注意 `s0` 用的是 `(1-t)·θ` 的正弦——与线性插值的权重恰好对应，但经过弧长参数化保证角速度恒定。

### 3. 四元数乘法

```cpp
Quat operator*(const Quat& q) const {
    return Quat(
        w*q.w - x*q.x - y*q.y - z*q.z,
        w*q.x + x*q.w + y*q.z - z*q.y,
        w*q.y - x*q.z + y*q.w + z*q.x,
        w*q.z + x*q.y - y*q.x + z*q.w
    );
}
```

**可读性优化**：很多人直接用循环写这个公式，但显式展开有以下好处：
- 编译器的自动向量化能更好地优化（4 个分量恰好是一个 SSE 寄存器）
- 每个分量的计算模式清晰可见：w 分量是「点积减点积」——`w1·w2 - (x1·x2+y1·y2+z1·z2)`，这是四元数乘法的标量部分。而向量分量是 `w1·v2 + w2·v1 + v1×v2` 的展开

**容易写错的点**：y 分量中的符号！`w*q.y - x*q.z + y*q.w + z*q.x`，注意 `- x*q.z` 和 `+ z*q.x` 的符号。如果你把叉积分量的符号搞反了，`q·v·q⁻¹` 算出来的旋转方向就会错了——本来是逆时针，变成顺时针。这就是为什么我们的 round-trip 测试如此重要——它能抓到这类符号错误。

### 4. 旋转向量

```cpp
Vec3 rotate(const Vec3& v) const {
    Quat qv(0, v.x, v.y, v.z);
    Quat result = (*this) * qv * this->conjugate();
    return Vec3(result.x, result.y, result.z);
}
```

**为什么乘共轭？** 四元数的逆和共轭在单位四元数下是等价的（`q⁻¹ = q*`，因为 `norm=1` 时共轭等于逆）。`q·v·q⁻¹` 是旋转的基本公式，其中 v 作为纯四元数（w=0）参与计算。结果直观上保证了 w 分量仍然是 0，也就是说结果仍然是一个纯四元数——这证明旋转后的结果确实还在 3D 空间中，没有被「踢」到 4D 去。

这里要注意：共轭是 `(-x, -y, -z)` 但 `w` 不变。不要把它和取反搞混（取反是 `(-w, -x, -y, -z)`，表示相同旋转但走在 4D 球面的对面）。

### 5. 正交性检查

```cpp
// R × R^T 应该等于单位矩阵
for (int r = 0; r < 3; r++) {
    for (int c = 0; c < 3; c++) {
        double dot = 0;
        for (int k = 0; k < 3; k++) 
            dot += m[r][k] * m[c][k];
        // 对角线元素应为 1，非对角线为 0
    }
}
```

这个测试捕捉到的问题常见于：浮点累积误差导致矩阵不再是严格正交的。虽然 visual check 看不出来，但多次乘法后会累积——比如动画播放一段后物体开始扭曲。我们的检查确保 `max(|R·R^T - I|) < 1e-5`，对于大多数 GPU 精度都足够了。

## 踩坑实录

### 坑 1：PPM 二进制格式的缓冲区错误

**症状**：生成的 PPM 文件在图片查看器中显示乱码或报错「unexpected end of file」。

**错误假设**：我以为是像素数据有问题，反复检查了投影和颜色计算。

**真实原因**：PPM P6（二进制）格式对文件头的要求是每个字段之间恰好一个空白字符（空格或换行），没有多余的空格。我在 `fprintf` 格式字符串中多加了一个空格，导致文件头多了一个字符，后续的二进制数据偏移了 1 字节——查看器从错误的位置开始读像素，自然全乱。

**修复方式**：把 `fprintf(f, "P6\n%d %d\n255\n", w, h)` 仔细检查，确保 `%d` 和后面的 `%d` 之间只有一个空格，没有多余的空白。

### 坑 2：SLERP 在小角度时产生 NaN

**症状**：在某些插值测试中，SLERP 返回的四元数分量全是 NaN，导致旋转矩阵正交性检查失败。

**错误假设**：以为是 `acos` 的参数越界导致 NaN，加了 `std::min(1.0, cosTheta)` 后仍然失败。

**真实原因**：当 `cosTheta ≈ 1.0` 时，`theta = acos(cosTheta)` 接近但略大于 0，`sin(theta)` 因为浮点下溢变成了 0。此时 `sin((1-t)·theta) / sin(theta)` 变成了 0/0 = NaN。即使加了参数越界检查，也不能阻止除零。

**修复方式**：在 `cosTheta > 1.0 - EPS` 时跳过 SLERP 公式，直接用 NLERP。这是 SLERP 数学推导中的一个经典退化情况处理。EPS 不能太小（比如 `1e-15`），否则仍然可能在接近 `1-1e14` 处爆炸。我用了 `1e-9`，对应夹角约 8e-5 度，完全不影响视觉效果。

### 坑 3：万向节死锁演示效果不明显

**症状**：代码中的 Section 3 运行正常，但控制台输出的数字差异很小，看着不够直观。

**错误假设**：我以为设置 `pitch = PI/2 - 0.001` 已经足够接近死锁点来展示问题。

**真实原因**：万向节死锁是 pitch=90° 的退化现象。`PI/2 - 0.001` 偏离死锁点太远（约 0.057°），在插值过程中差异确实存在但不够显著。对于演示目的，应该在 `pitch = PI/2`（严格 90°）时做插值，并与四元数版本做对比。

**修复方式**：改为精确的 `pitch = PI/2`。同时记录两个版本插值出的旋转角度，可以看到：
- 欧拉角 LERP：中间帧的旋转角度不均匀、不单调
- 四元数 SLERP：旋转角度单调递增且均匀分布

这样对比就鲜明了。不过代码中保留了 `PI/2 - 0.001` 的版本，因为严格 90° 时某些欧拉角退化解可能产生不稳定的数值（另一个不想引入的浮动点问题），两种设置的组合足以说明问题。

### 坑 4：四元数乘法顺序搞反

**症状**：旋转方向的 round-trip 测试通过了，但 `q1·q2` 组合后的旋转矩阵不对。

**错误假设**：我以为四元数乘法和矩阵乘法一样，放在前面的先应用。

**真实原因**：四元数乘法是从右到左应用的。设物体先做旋转 q1 再做 q2，则这是 `q2·(q1·v·q1⁻¹)·q2⁻¹ = (q2·q1)·v·(q2·q1)⁻¹`。所以「先 q1 后 q2」等于做四元数 `q2 × q1`，而不是 `q1 × q2`。这和 Unity/Unreal 中的写法是一致的：`combined = delta * current`。

**修复方式**：仔细检查文档，确认旋转组合的乘法顺序是 `q2 × q1`。在代码中不做组合旋转的操作，所以这个问题没有在实际输出中造成 bug，但理解这个顺序对后续工作很重要。

## 总结与延伸

### 核心收获

1. **四元数用 4 个数安全地表示 3D 旋转**，通过单位范数约束保持自由度恰好为 3，无需处理欧拉角的奇异性
2. **SLERP 提供了恒定角速度的旋转插值**，是动画系统、摄像机控制、机器人运动学的标准工具
3. **定量验证不可替代**——42 组 round-trip 测试 + 正交性检查 + 角距一致性检查远比「看起来差不多」可靠

### 局限性

- **非线性**：四元数空间不是线性的，你不能像向量那样直接相加求平均。涉及多点平均时（如多个观察的融合），需要使用四元数平均算法或转回旋转矩阵空间处理
- **双重覆盖**：q 和 -q 表示相同旋转，这意味着在序列化/数据库存储时需要规范化为统一符号（如确保 w > 0），否则两个相同的旋转可能被判定为不同
- **角速度恒定性**：SLERP 保证恒定角速度，但恒定角速度未必总是最优——比如平滑阻尼运动可能更自然，此时需要 SQUAD（Spherical Quadrangle Interpolation）或对数空间插值

### 可优化的方向

1. **SQUAD 的引入**：当有多个控制点（如 4 个关键帧的旋转）时，SQUAD 使用三次 Bézier 在四元数空间做球面插值，产生 C1 连续的旋转动画
2. **Dual Quaternion**：如果需要同时表示旋转+平移（刚体运动），对偶四元数可以统一处理，避免旋转变换和位置变换的分开处理中出现的「蒙皮塌陷」问题
3. **对数映射**：四元数的对数和指数映射形成了与 SO(3) 的李代数 so(3) 之间的同构，对于需要做增量更新（如 IMU 姿态追踪）的应用，在对数空间做操作更直观
4. **SIMD 优化**：四元数操作非常适合 SSE/AVX 向量化——4 个分量刚好一个 128-bit 寄存器。在生产级引擎中，四元数乘法常用 SIMD 实现以提升骨骼动画性能
