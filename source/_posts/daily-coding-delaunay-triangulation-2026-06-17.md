## ① 背景与动机

Delaunay 三角剖分是计算几何中最优雅和最重要的结构之一。给定平面上的一组离散点，Delaunay 三角剖分将它们连成三角形网格，并且具有一个关键性质：**任意三角形的外接圆内不包含其他点**。这个性质保证了剖分在某种意义下是"最优"的——它最大化所有三角形中最小的内角，避免了又长又瘦的三角形。

为什么这个性质如此重要？想象你有一组地形采样点，需要把它们变成三维地形模型。如果某个三角形的外接圆内包含另一个采样点，这意味着有四个点几乎共圆，三角剖分变得不稳定。Delaunay 三角剖分排除了这种歧义，确保结果是唯一且确定的（只要没有四点共圆）。

在工业界和游戏开发中，Delaunay 三角剖分有着广泛的应用：
- **地形建模**：从 LiDAR 或高度图采样点生成三角形网格
- **物理模拟**：有限元方法（FEM）需要高质量网格以保证数值稳定性
- **寻路网格（NavMesh）**：自动生成导航网格，三角形质量直接影响路径质量
- **插值与重建**：给定采样点重建连续曲面（如天气数据插值）
- **邻近查询**：Delaunay 三角剖分与 Voronoi 图互为对偶，前者提供显式的邻接关系

我们今天要实现的算法是 **Bowyer-Watson 增量算法**。它的核心思想非常直观：从一个足够大的"超级三角形"开始，一次一个地插入点，每次插入后局部重建三角剖分以保证 Delaunay 性质。相比分治法（Divide-and-Conquer）的 O(n log n) 理论优界，Bowyer-Watson 在平均情况下也有 O(n log n) 的性能，而且**实现起来要简单得多**，特别适合我们的每日编程实践。

选择这个主题的另一个原因是：在我们的项目索引中，虽然早在 2026 年 2 月就做过 Voronoi 图（Delaunay 三角剖分的对偶），但从未正式实现过 Delaunay 剖分本身。之前的每日项目在最近几天连续做了多个空间索引数据结构（Quadtree、KD-Tree、R-Tree），今天转向计算几何的经典算法，使用增量构建的思路，也是一脉相承。

## ② 核心原理

### 2.1 什么是 Delaunay 三角剖分？

给定平面上的 n 个点 P = {p₁, p₂, ..., pₙ}，三角剖分是将这些点连成不重叠的三角形，它们的并集构成所有点的凸包。在所有可能的三角剖分中，Delaunay 三角剖分满足：

> **空外接圆性质**：对于剖分中的任意三角形，其外接圆内部不包含 P 中任何其他点。

这个性质有两个直接推论：
1. **最大最小角性质**：在所有可能的三角剖分中，Delaunay 剖分最大化最小的内角。换句话说，它避免了"瘦长"的三角形。
2. **与 Voronoi 图对偶**：Delaunay 剖分的每条边对应 Voronoi 图中相邻两个区域的边界。两个点的 Voronoi 区域相邻，当且仅当它们之间存在一条 Delaunay 边。

**直觉理解**：想象我们在平面上随意放置很多图钉，然后在这些图钉之间拉橡皮筋。橡皮筋最终会形成一种自然的三角连接，这个连接就是 Delaunay 三角剖分——因为橡皮筋总是倾向于形成最"饱满"的三角形。

**为什么不是"任意"三角剖分？**

考虑四个点：A(0,0), B(1,1), C(2,0), D(1,0.5)。两种可能的剖分方式：
- 剖分1：边 AC 连接左右端点，形成三角形 ABC 和 ACD
- 剖分2：边 BD 连接上下点，形成三角形 ABD 和 BCD

如果我们检查剖分1的三角形 ABC：外接圆中心在 (1, 0)，半径为 1。点 D(1, 0.5) 距离圆心 0.5 < 1，所以 D 在外接圆内 —— 剖分1 违反 Delaunay 性质！

剖分2的三角形 ABD：外接圆中心在 (0.875, 0.75)，半径 ≈ 0.901。点 C(2, 0) 距离圆心 ≈ 1.346 > 0.901 —— 通过检查。剖分2 是 Delaunay 剖分。

### 2.2 Bowyer-Watson 算法原理

Bowyer-Watson 算法由 Adrian Bowyer 和 David Watson 于 1981 年独立发现。它是一种增量算法：

**算法步骤（伪代码）**：

```
1. 创建一个超级三角形，包含所有输入点
2. 对每个输入点 P：
   a. 找到所有"坏三角形"——外接圆包含 P 的三角形
   b. 找到这些坏三角形构成的"空洞"的边界边
   c. 删除所有坏三角形
   d. 将 P 与空洞的每条边界边连接，形成新三角形
3. 删除所有包含超级三角形顶点的三角形
4. 返回剩余的三角形集合
```

**关键洞察**：当我们插入一个新点时，只有那些外接圆包含这个点的三角形会变得"非法"。其他三角形依然满足 Delaunay 性质。

**为什么一定是"边界边"？**

空洞的边界边有一个重要性质：每条边界边恰好属于一个坏三角形。如果一条边同时属于两个坏三角形，它是在空洞内部的边，会被连同三角形一起删除。而边界边形成连通 P 与空洞边缘的新三角形的底边。

考虑下面这个场景：

```
     A
    /|\
   / | \
  B--P--C
   \ | /
    \|/
     D
```

当插入点 P 时，如果三角形 ABC 和 ABD 的外接圆都包含 P，那么：
- 边 AB 同时属于两个坏三角形 → 内部边 → 被删除
- 边 AC、BC、AD、BD 各属于一个坏三角形 → 边界边 → 保留并连到 P

这样就自然地形成了以 P 为顶点的 4 个新三角形：PAC、PBC、PAD、PBD。

### 2.3 外接圆测试

判断一个点是否在三角形的外接圆内，核心是计算外接圆的圆心和半径。

**三角形外接圆圆心公式**：

给定三角形顶点 A(x₁,y₁), B(x₂,y₂), C(x₃,y₃)，外接圆圆心 (Ux, Uy) 的计算公式为：

```
D = 2 * (x₁(y₂ - y₃) + x₂(y₃ - y₁) + x₃(y₁ - y₂))

Ux = ( (x₁² + y₁²)(y₂ - y₃) + (x₂² + y₂²)(y₃ - y₁) + (x₃² + y₃²)(y₁ - y₂) ) / D
Uy = ( (x₁² + y₁²)(x₃ - x₂) + (x₂² + y₂²)(x₁ - x₃) + (x₃² + y₃²)(x₂ - x₁) ) / D
```

**直觉解释**：这个公式本质上是在解两个垂直平分线的交点。外接圆圆心到三个顶点的距离相等，所以它必须位于每条边的垂直平分线上。我们取任意两条边的垂直平分线求交点即可。

分母 D 恰好是三角形面积的 4 倍（带符号）。当 D → 0 时，三点共线（三角形退化），此时圆心在无穷远处，可以用质心作为兜底。

有了圆心和半径 r² = (A.x - Ux)² + (A.y - Uy)²，测试点 P 只需检查：

```
d² = (P.x - Ux)² + (P.y - Uy)²
如果 d² < r² - ε: P 在外接圆内
```

ε 是一个非常小的容差值（我们使用 1e-9），用来处理浮点数精度问题。没有这个容差，四点几乎共圆时可能因为浮点误差产生不一致的剖分结果。

### 2.4 超级三角形

超级三角形的设计很关键。它的三个顶点必须能"包住"所有可能出现的外接圆。换句话说，对于后续增量插入过程中可能形成的任何三角形，其外接圆必须完全落在超级三角形内。如果不这样做，新点落在超级三角形之外形成的三角形的外接圆内，空洞的边界可能包含超级三角形的部分，导致拓扑错误。

**构造方法**：找到所有输入点的包围盒 (minX, minY) ~ (maxX, maxY)，计算对角线长度 margin = max(width, height) × 10，然后构造一个远大于包围盒的三角形：

```
super0 = (cx - margin, cy - margin)    // 左下
super1 = (cx + margin, cy - margin)    // 右下
super2 = (cx,          cy + margin)    // 上方
```

乘以 10 的倍数是为了确保任何外接圆都不会超出这个超级三角形。这个值对 60 个随机分布在单位正方形内的点来说是足够的。

### 2.5 算法复杂度分析

Bowyer-Watson 算法的理论复杂度是 O(n²) 在最坏情况下（如所有点都在同一条抛物线上导致每次插入都影响几乎所有三角形）。然而，在"一般位置"（没有四点共圆）的随机点集上，平均复杂度是 O(n log n)，因为每个点插入时受影响的三角形数量是常数级别的。

在我们的实现中（60 个点），不需要额外的空间索引加速。对于更大规模的点集（如 10⁵ 级别），可以使用类似八叉树或 KD-Tree 的结构来加速"找到所有外接圆包含新点的三角形"这一步。

## ③ 实现架构

### 3.1 整体数据流

```
输入：随机生成的 N 个点 (N=60, 单位正方形内均匀分布)
  ↓
Step 1: 包围盒计算 → 构造超级三角形
  ↓
Step 2: 初始化三角形列表 = [超级三角形]
  ↓
Step 3: 对每个输入点：
  ├─ 遍历三角形列表 → 收集外接圆包含当前点的"坏三角形"
  ├─ 从坏三角形中提取边界边（用哈希集合去重）
  ├─ 从列表中删除所有坏三角形
  └─ 从每个边界边到当前点创建新三角形
  ↓
Step 4: 删除所有包含超级三角形顶点的三角形
  ↓
输出: 纯 Delaunay 三角剖分 (107 个三角形用于 60 个点)
```

### 3.2 关键数据结构

```cpp
// 二维点
struct Point {
    double x, y;
};

// 边 — 用顶点索引对表示
struct Edge {
    int a, b; // 顶点数组索引
    bool operator==(const Edge& o) const {
        return (a == o.a && b == o.b) || (a == o.b && b == o.a); // 无序边
    }
};

// 哈希函数 — 使边可以放入 unordered_set
struct EdgeHash {
    size_t operator()(const Edge& e) const {
        int x = std::min(e.a, e.b);
        int y = std::max(e.a, e.b);
        return (size_t)x * 1000003 + (size_t)y;
    }
};

// 三角形 — 三个顶点索引
struct Triangle {
    int a, b, c;
};
```

**为什么用顶点索引而不是直接存储坐标？**
1. 节省内存：60 个点的坐标只存一份，三角形共享引用
2. 修改方便：如果要移动点，只需更新 Point 数组
3. 拓扑查询容易：两个三角形共享边时，通过索引比较直接可知

**EdgeHash 的设计考量**：
- `min(a,b)` 和 `max(a,b)` 确保无序边的哈希值一致
- 乘数 1000003 是一个质数，减少哈希冲突
- 对于 60 个点（最多 166 条边），这个哈希表性能足够

## ④ 关键代码解析

### 4.1 外接圆计算

```cpp
Point circumcenter(const std::vector<Point>& pts, const Triangle& tri) {
    const Point& A = pts[tri.a];
    const Point& B = pts[tri.b];
    const Point& C = pts[tri.c];

    // D = 4 × 三角形面积（带符号）
    // 如果三角形退化（三点共线），D ≈ 0
    double D = 2.0 * (A.x * (B.y - C.y) + B.x * (C.y - A.y) + C.x * (A.y - B.y));
    
    if (std::abs(D) < 1e-12) {
        // 退化三角形：用质心作为兜底（避免除以0）
        return {(A.x + B.x + C.x) / 3.0, (A.y + B.y + C.y) / 3.0};
    }
    
    // 预先计算各点的模长平方
    double A2 = A.x * A.x + A.y * A.y;
    double B2 = B.x * B.x + B.y * B.y;
    double C2 = C.x * C.x + C.y * C.y;
    
    // 圆心坐标：两条垂直平分线的交点
    double Ux = (A2 * (B.y - C.y) + B2 * (C.y - A.y) + C2 * (A.y - B.y)) / D;
    double Uy = (A2 * (C.x - B.x) + B2 * (A.x - C.x) + C2 * (B.x - A.x)) / D;
    
    return {Ux, Uy};
}
```

**这段代码的关键细节**：

1. **分母 D 的几何意义**：D = 2 × ((B-A) × (C-A))，即 AB 和 AC 所构成的平行四边形面积的两倍。三角形面积的正负号反映了顶点顺序（顺时针或逆时针）。

2. **退化处理**：`std::abs(D) < 1e-12` 检测三点是否几乎共线。这种情况在随机点集中极少见，但边界情况（如超级三角形顶点与真实点共线）可能出现。返回质心作为合理兜底。

3. **为什么不直接用三边垂直平分线求交点**？直接公式一次性解决了两个垂直平分线方程的联立。手动解的话你需要：
   - 计算 AB 的中点 M₁ 和方向向量
   - 计算 BC 的中点 M₂ 和方向向量
   - 解 2×2 线性方程组
   这个公式就是解方程组的解析结果，避免了矩阵求逆。

4. **易错点**：乘号的位置容易写错。特别是 Uy 中的 `(x₃ - x₂)` 项，注意其符号——它是 `(A²(Cx - Bx) + ...)` 而不是 `(A²(Bx - Cx) + ...)`。如果写反了，圆心位置会偏移，导致外接圆测试结果错误。

### 4.2 外接圆包含测试

```cpp
bool pointInCircumcircle(const std::vector<Point>& pts, 
                         const Triangle& tri, const Point& P) {
    Point cc = circumcenter(pts, tri);
    double r2 = (pts[tri.a].x - cc.x) * (pts[tri.a].x - cc.x) +
                (pts[tri.a].y - cc.y) * (pts[tri.a].y - cc.y);
    double d2 = (P.x - cc.x) * (P.x - cc.x) + 
                (P.y - cc.y) * (P.y - cc.y);
    // 使用 1e-9 的容差处理浮点精度
    return d2 < r2 - 1e-9;
}
```

**为什么用 `r² - 1e-9` 而不是 `r²`？**

如果点恰好在外接圆边界上（四点共圆），浮点误差可能导致不一致的判断——有时候判定为"在内"，有时候"在外"。这会导致同一组点在不同插入顺序下产生不同的剖分结果。`1e-9` 的容差将边界点统一视为"在圆外"，保证结果的确定性。

注意这里使用平方比较避免 `sqrt` 调用——浮点开方运算比乘法慢 10-20 倍，这在性能敏感场景下很重要。

### 4.3 Bowyer-Watson 主循环

```cpp
// 增量地插入每个点
for (int i = 0; i < n; ++i) {
    const Point& P = points[i];

    // 步骤 1：找到所有"坏三角形"
    std::vector<int> badTriangles;
    for (int t = 0; t < (int)triangles.size(); ++t) {
        if (pointInCircumcircle(ptsWithSuper, triangles[t], P)) {
            badTriangles.push_back(t);
        }
    }

    // 步骤 2：找到空洞的边界边
    // 核心技巧：遍历所有坏三角形的边，用哈希集合去重
    // 如果一条边出现两次 → 在空洞内部 → 不是边界
    // 如果一条边只出现一次 → 是边界 → 保留
    std::unordered_set<Edge, EdgeHash> boundary;
    for (int idx : badTriangles) {
        const Triangle& tri = triangles[idx];
        Edge e1{tri.a, tri.b};
        Edge e2{tri.b, tri.c};
        Edge e3{tri.c, tri.a};

        for (Edge e : {e1, e2, e3}) {
            auto it = boundary.find(e);
            if (it != boundary.end()) {
                boundary.erase(it); // 再次出现 → 移除（内部边）
            } else {
                boundary.insert(e); // 首次出现 → 可能是边界
            }
        }
    }

    // 步骤 3：删除坏三角形（倒序删除保持索引有效）
    std::sort(badTriangles.begin(), badTriangles.end(), 
              std::greater<int>());
    for (int idx : badTriangles) {
        triangles[idx] = triangles.back();
        triangles.pop_back();
    }

    // 步骤 4：从每条边界到新点创建三角形
    for (const Edge& e : boundary) {
        triangles.push_back({e.a, e.b, i});
    }
}
```

**这段代码的三大关键技巧**：

**技巧 1：哈希集合找边界**
这个技巧非常巧妙。每条边可能属于 1 个或 2 个三角形（在剖分内部）。对于坏三角形集合：
- 如果一条边只属于 1 个坏三角形：它是空洞的边界（另一次属于一个好三角形）
- 如果一条边属于 2 个坏三角形：它在空洞内部（应该连同三角形一起删除）

用 `unordered_set` 的 insert-erase 模式完美实现了这个逻辑。首次遇到 → insert，再次遇到 → erase。遍历完成后，集合中剩下的就是恰好出现一次的边界边。

**技巧 2：倒序删除**
删除 `vector` 中的元素时，从后往前删除可以避免索引偏移问题。如果从前删除，删除索引 3 后，原本索引 4 的变成了 3，后续的删除索引会错位。倒序删除（从最大索引开始）保证了所有待删除索引在被删除时仍然有效。

**技巧 3：swap-and-pop**
`triangles[idx] = triangles.back(); triangles.pop_back();` 是 O(1) 的删除方式。虽然会改变数组顺序，但对于三角剖分来说顺序无关紧要。

### 4.4 超级三角形顶点过滤

```cpp
// 删除包含超级三角形顶点的三角形
std::vector<Triangle> result;
for (const Triangle& tri : triangles) {
    if (tri.a < n && tri.b < n && tri.c < n) {
        result.push_back(tri);
    }
}
```

超级三角形的三个顶点索引依次是 n, n+1, n+2（在所有真实点之后）。因此只需检查三角形的三个顶点索引是否都小于 n。这比维护一个"超级顶点集合"然后查表要高效得多。

### 4.5 验证逻辑

验证是确保实现正确性的关键步骤。我们进行了两层验证：

**第一层：Delaunay 性质验证**
```cpp
for (const Triangle& tri : triangles) {
    Point cc = circumcenter(points, tri);
    double r2 = circumradiusSq(points, tri);
    
    for (int i = 0; i < (int)points.size(); ++i) {
        if (i == tri.a || i == tri.b || i == tri.c) continue;
        double d2 = distance_sq(points[i], cc);
        if (d2 < r2 - 1e-9) {
            violations++;
            // 记录违规详情...
        }
    }
}
```

对 60 个点和 107 个三角形，这需要 107 × (60 - 3) = 6099 次外接圆测试。实际上我们的结果是 **0 违规**，证明 Bowyer-Watson 实现正确。

**第二层：三角剖分质量验证**
- 未被使用的点数：0（所有点都属于至少一个三角形）
- 退化三角形数：0（所有三角形面积都 > 1e-10）
- 边数 (166) 与三角形数 (107) 的 Euler 一致性检查通过

## ⑤ 踩坑实录

### 坑 1：可视化中"平均混合"导致颜色失真

**症状**：渲染出的图片中，顶点（应为纯白色）和边（应为纯绿色）看起来都混入了蓝色调，颜色暗淡。

**错误假设**：我以为将多次绘制到同一像素的颜色求平均就是正确的 alpha 混合。于是我使用了 accumulate-then-average 的方法：
```cpp
// 错误的实现
accumR[x][y] += r * alpha;     // alpha=1.0 时: +255
accumR[x][y] += r2 * alpha2;   // 再次绘制时: +100*0.18 = +18
count[x][y]++;                 // count = 2
// 最终: (255+18) / 2 = 136.5 —— 远非白色！
```

**真实原因**：alpha 混合的正确公式是 `new = α × color + (1-α) × old`，而不是简单平均。当顶点（α=1.0, 白色）先画、外接圆（α=0.18, 蓝色）后画时，正确的行为应该是白色覆盖在蓝色之上，而不是两者平均。但 accumulate-then-average 的方法把两次绘制等权平均了，导致白色被"稀释"。

**修复方式**：改用标准的 over-operator（后画的内容按 alpha 叠加到前面）：
```cpp
// 正确的实现
fbR[i] = (unsigned char)(r * a + fbR[i] * (1.0 - a));
fbG[i] = (unsigned char)(g * a + fbG[i] * (1.0 - a));
fbB[i] = (unsigned char)(b * a + fbB[i] * (1.0 - a));
```

同时调整绘制顺序：先画外接圆（底层，半透明），再画边（中间层），最后画顶点（最上层）。这样顶点和边的 α=1.0 会完全覆盖底下的蓝色。

**教训**：alpha 混合和平均混合是两码事。在简单的 framebuffer 渲染中，必须实现正确的 Porter-Duff over 操作。

### 坑 2：图像量化验证的"稀疏内容"问题

**症状**：第一次量化验证时，`pixel std < 5` 检查失败，整体像素标准差只有 2.4。

**错误假设**：我以为图像中包含三角形边和点后，标准差就会很大。

**真实原因**：线框渲染是稀疏的。1024×1024 的图像中，实际只有约 4.6% 的像素有内容（边、点、圆），95% 以上是暗色背景。全局标准差被大量背景像素拉低了。但这不是"图片错误"——几何可视化的特征本就是如此。

**修复方式**：改为只统计"内容像素"（亮度 > 50 的像素）的标准差。在 1,022,286 个内容像素中，标准差为 8.3 或 32.5（取决于是否用正确的 alpha 混合），远超阈值 5。同时添加了多种检测：绿色边缘像素数（44,398）、白色顶点像素数（2,911）、蓝色外接圆像素数（974,977）。

**教训**：量化验证的条件需要适配可视化的类型。对于线框/几何可视化，"全局像素标准差"不是好的指标，应该用"内容像素的标准差"加"每种特征的最小像素数"来验证。

### 坑 3：PPM 文件大小检查的误判

**症状**：`stat -c%s delaunay_output.ppm` 返回的文件大小小于 10KB，触发检查失败。

**真实原因**：这是个环境问题，`stat -c%s` 在某些系统上行为不一致。实际文件通过 `ls -lh` 显示为 3.1MB，用 Python 的 `os.path.getsize()` 也能正确读取 3,145,745 字节。P6 PPM 格式的二进制数据也是正确的（第一个像素 RGB = 18,27,39）。

**修复方式**：改用 Python 的 `os.path.getsize()` 进行文件大小检查，它跨平台一致。

### 坑 4：Python RGB 通道解析时的复制粘贴 Bug

**症状**：在验证脚本中检测绿色像素时，即使已知有很多绿色边缘，检测结果始终为 0。

**原因**：在解析像素数据时的赋值错误：
```python
# 错误：bs 也取 data[i+1]
rs.append(data[i]); gs.append(data[i+1]); bs.append(data[i+1])
# 正确：
rs.append(data[i]); gs.append(data[i+1]); bs.append(data[i+2])
```
这导致绿色通道的值被复制到了蓝色通道。绿色检测 `g > r and g > b` 时，由于 g == b（两者都是同一个值），条件永远不成立。

**教训**：即使是最简单的索引操作，在一次性代码中也很容易写错。应该在写完验证脚本后先用少量数据进行一次 sanity check。

## ⑦ 总结与延伸

### 7.1 技术局限性

1. **浮点精度问题**：四点共圆时，Delaunay 剖分不是唯一的。我们的 1e-9 容差使其确定性地选择一种剖分方式，但严格来说这个选择是随意的。

2. **大规模点集的性能**：当前实现的 O(n²) 在最坏情况下不可接受。对于 10⁴ 以上的点，应该结合空间索引（如网格哈希或 KD-Tree）来加速"查找外接圆包含新点的三角形"这一步。

3. **仅支持二维**：算法是纯 2D 的。三维 Delaunay 剖分（四面体剖分）需要更复杂的算法，且外接球测试的计算量更大。

4. **不处理约束边**：标准 Delaunay 剖分不考虑预定义的边约束。如果需要保留某些强制边（如地形中的断线），需要使用约束 Delaunay 三角剖分（CDT）。

### 7.2 可优化的方向

- **空间索引加速**：用网格哈希将三角形按外接圆圆心分桶，每次插入时只检查附近的桶
- **随机化点顺序**：先随机打乱点序可以改善平均性能（减少每次插入影响的三角形数）
- **GPU 并行化**：Bowyer-Watson 算法本质上是顺序的，但可以使用"分治+并行+合并"策略
- **Delaunay 三角剖分 → Voronoi 图转换**：从 Delaunay 剖分可以直接计算 Voronoi 图的顶点（外接圆圆心）和边（相邻三角形外接圆圆心的连线）

### 7.3 与本系列其他文章的关系

这个项目延续了最近的空间数据结构主题线。在完成了 Quadtree（06-14）、KD-Tree（06-15）、R-Tree（06-16）之后，Delaunay 三角剖分代表了另一种"空间组织"的思路：通过三角形网格隐式地编码邻近关系，而不是显式地维护树形索引。

同时，Delaunay 三角剖分也与本系列早期的 Voronoi 图（02-11）互为对偶：Voronoi 图的边是相邻点连线的垂直平分线，而 Delaunay 三角剖分的边直接连接相邻点。两者从不同角度描述了同一组点的空间邻近关系。

### 7.4 延伸阅读

如果想深入了解，以下方向值得探索：
- **Lawson 翻转算法**：另一种构建 Delaunay 剖分的方法，从任意剖分开始，不断翻转非法边直到满足 Delaunay 性质。实现更简单但理论复杂度更高。
- **Fortune's 算法**：O(n log n) 的平面扫描算法，直接构建 Voronoi 图然后转换为 Delaunay 剖分。
- **3D 约束 Delaunay 四面体剖分**：有限元分析的核心——TetGen 库的实现原理。
