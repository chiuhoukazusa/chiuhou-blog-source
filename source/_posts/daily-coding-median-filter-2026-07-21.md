## 背景与动机

### 什么是椒盐噪声？

在图像采集、传输或存储过程中，传感器故障、信道干扰或存储介质损坏会以不可预测的方式污染像素值。其中一种极具破坏性的噪声类型就是 **椒盐噪声（Salt & Pepper Noise）**，也被称为 **脉冲噪声（Impulse Noise）**。

椒盐噪声的特点非常鲜明：
- **盐噪声（Salt）**：随机的像素点被置为纯白（255, 255, 255），就像撒了盐粒
- **胡椒噪声（Pepper）**：随机的像素点被置为纯黑（0, 0, 0），就像撒了黑胡椒
- 这些污染像素与周围像素值完全不同，在图像上形成刺眼的黑白斑点

对于一张 512×512 的图像，哪怕只有 5% 的像素被污染，就有超过 13,000 个像素点变成纯黑或纯白。这样的图像在视觉上几乎无法接受，更不用说后续的图像分析、目标检测或模式识别任务了。

### 为什么线性滤波器不行？

直觉上，我们可能会觉得"模糊一下就好了"——毕竟把噪声像素和周围的好像素平均一下，噪声的影响应该会被稀释。这就是 **高斯模糊（Gaussian Blur）** 或 **均值滤波（Mean Filter）** 的思路。但这恰恰是问题所在。

来看一个具体例子。假设有一个 3×3 的窗口，中心像素被椒盐噪声污染变成了 255。周围的 8 个像素值是 `[50, 52, 51, 49, 53, 50, 48, 52]`：

- **均值滤波**：计算结果 = (255 + 50+52+51+49+53+50+48+52)/9 ≈ **73.3**
- **中值滤波**：排序后取中间值 = **51**

真相是什么？周围像素的平均值是 50.6，所以真实值应该在 50 左右。均值 73.3 被那个 255 严重污染了（偏离真实值 ~47%），而中值 51 几乎完全不受影响。

**这就是中值滤波的核心优势**：对于脉冲噪声这类极端值（outlier），中位数天然免疫，而均值则会被严重拖累。

从统计学角度看，中位数是一个 **稳健统计量（Robust Statistic）**，它的 **崩溃点（Breakdown Point）** 是 50%——在一个窗口中，哪怕有接近一半的数据点是噪声，中位数仍然能给出合理的结果。相比之下，均值的崩溃点是 0%，一个极端值就能彻底破坏它。

### 工业界的实际应用

中值滤波在工业界有广泛的实际应用，绝不只是课堂上的玩具算法：

**1. CCD/CMOS 传感器死像素修复**：数码相机的图像传感器（CMOS 或 CCD 芯片）上可能存在制造缺陷导致的"死像素"——某些感光单元始终输出最大值（白色死像素）或最小值（黑色死像素）。相机的图像处理管线（ISP Pipeline）在 RAW 数据阶段就会运行中值滤波来修复这些死像素。

**2. 医学影像去噪**：在超声成像、X 光片和 MRI 中，椒盐噪声经常出现。医学影像的特殊性在于——你**绝对不能**模糊掉重要的诊断细节（比如微小的肿瘤边界或血管边缘）。中值滤波在去除噪声的同时保持边缘清晰，是医学影像预处理的标配工具。

**3. 卫星与遥感图像**：卫星图像传输过程中，大气干扰和信号衰减会产生随机脉冲噪声。在土地利用分类、目标检测等任务之前，中值滤波是预处理 Pipeline 的第一步。

**4. 视频监控系统**：低光照条件下的监控摄像头会产生大量椒盐噪声。中值滤波算法简单（不需要浮点运算），可以在嵌入式硬件上实时运行。

**5. 图像修复的传统方法**：在深度学习出现之前，中值滤波及其变体（如自适应中值滤波）是修复老照片划痕、污点的主要手段。

### 本文目标

1. **生成测试图像**：包含均匀区域、边缘、几何形状的受控场景
2. **添加椒盐噪声**：可配置密度的可控噪声注入
3. **实现中值滤波**：支持可配置窗口大小的滑动窗口中值滤波器
4. **实现高斯模糊**：作为线性滤波器的对比基准
5. **量化验证**：使用 PSNR、噪声移除率、边缘保持指数、均匀区域方差等指标进行严格量化评估

## 核心原理

### 数学定义

对于一个位于坐标 (x, y) 的像素，给定大小为 w×w（w 为奇数）的滑动窗口 W，中值滤波的输出定义为：

```
I_median(x, y) = median{I(i, j) | (i, j) ∈ W(x, y)}
```

median 算子将窗口内的所有像素值按升序排列，取中间位置的值。

**直觉解释**：在像素 (x,y) 的周围 w×w 邻域内，收集所有像素值，排序，取中间那个。因为椒盐噪声总是取极端值（0 或 255），在排序后它们会自动跑到数组的两端，而中位数永远不会被选中——除非窗口中超过一半的像素都是噪声。

### 为什么中位数免疫极端值？

用一个更严谨的例子说明。假设 3×3 窗口有 9 个像素值：

```
{50, 52, 51, 49, 255, 53, 50, 48, 52}
```

排序后：
```
{48, 49, 50, 50, 51, 52, 52, 53, 255}
```

中位数是第 5 个（索引 4）：**51**。

敏感性分析：
- 如果把 255 改成 1000（更极端的异常值），排序后还是最后一个，中位数还是 **51**
- 如果把 255 改成 0（方向相反的异常值），排序后跑到第一个，中位数还是 **51**

这就是"稳健统计量"的含义：**只要异常值不超过窗口大小的一半，中位数就完全不受它们的影响。**

均值呢？
```
mean = (50+52+51+49+255+53+50+48+52)/9 = 73.3
```
如果异常值变成 1000：
```
mean = 156.1
```
**均值完全被异常值拖着走。**

### 中值滤波 vs 高斯模糊

| 维度 | 中值滤波 | 高斯模糊 |
|------|---------|----------|
| 滤波器类型 | 非线性（Non-linear） | 线性（Linear） |
| 操作 | 排序取中位数 | 加权求和（卷积） |
| 对脉冲噪声 | 几乎完全去除 | 噪声像素会扩散到邻域 |
| 边缘保持 | 优秀 | 差（边缘会被模糊） |
| 计算复杂度 | O(w²×n×log(w²)) | O(w²×n) |
| 崩溃点 | 50% | 0% |

**核心差异的本质**：

高斯模糊的卷积核是一个固定的加权平均模板（中心权重最高，向四周递减），它对图像的所有像素进行同样的"软化"操作：
- 在**均匀区域**：噪声像素被周围的好像素"稀释"，但没有被完全消除——噪声的贡献虽然降低了但依然存在
- 在**边缘区域**：边缘两侧的不同颜色值被加权平均后混合在一起，导致边缘模糊

中值滤波则完全不一样：
- 在**均匀区域**：纯黑和纯白的噪声像素在排序后会被推到两端，中位数一定是该区域的主色调
- 在**边缘区域**：如果窗口中心靠近边缘，窗口中大部分像素仍然是边缘一侧的颜色，中位数也来自这一侧——边缘得以保持

### 窗口大小的影响

- **3×3 窗口**：最温和，去噪能力强（15% 密度噪声下移除率可达 99.7%），细节保持最好
- **5×5 窗口**：更强去噪能力（移除率 99.9%），但会抹去小细节
- **7×7+ 窗口**：适合极高噪声密度，但图像明显柔化

### 计算复杂度与优化

朴素实现：O(H × W × w² × log(w²))。优化技巧：

1. **`std::nth_element` 替代完整排序**：QuickSelect 算法，平均 O(n)，比完整排序 O(n log n) 快
2. **增量直方图更新**：大窗口下维护直方图，每移动一个像素只更新一列
3. **可分离实现不适用**：中值滤波不是可分离的，不能像高斯那样拆成两个 1D pass

## 实现架构

### 整体数据流

```
Test Image Generator → Clean Image → +Salt&Pepper(15%) → Noisy Image
                                                              │
                                          ┌───────────────────┤
                                          ▼                   ▼
                                    Median Filter       Gaussian Blur
                                    (3×3, 5×5)         (σ=0.8,1.5,2.5)
                                          │                   │
                                          ▼                   ▼
                                      Filtered.ppm         Blurred.ppm
                                          │                   │
                                          └─────────┬─────────┘
                                                    ▼
                                          Quantitative Verification
                                          (PSNR/NoiseRm/Edge/Variance)
```

### 图像数据结构

```cpp
struct Image {
    int w, h;
    std::vector<uint8_t> r, g, b;  // 分离的三个颜色通道
};
```

三个关键设计决策：
1. **分离 R/G/B 通道**：每个通道独立处理，连续内存布局对缓存友好
2. **使用 `uint8_t`**：像素值 [0, 255]，8 位整数是最自然高效的选择
3. **不用浮点数**：中值滤波只需比较和排序，整数运算更快

### 高斯模糊的可分离实现

高斯核的重要性质：
```
G(x, y, σ) = G(x, σ) × G(y, σ)
```

2D 高斯可以分解为两个 1D 高斯：
```
I * G_2D = I * G_x * G_y
```

计算量从 O(w²) 降到 O(2w)，这也是 GPU 高斯模糊的实现方式（horizontal blur pass + vertical blur pass）。

## 关键代码解析

### 1. 椒盐噪声生成器

```cpp
Image addSaltPepper(const Image& src, double density) {
    Image out = src.clone();                          // ① 复制干净图像
    int total = out.w * out.h;
    int noisePixels = (int)(total * density);         // ② 计算污染像素数量
    srand(12345);                                      // ③ 固定种子，保证可复现

    for (int i = 0; i < noisePixels; i++) {
        int idx = rand() % total;                      // ④ 随机选择像素
        // ⑤ 随机盐(255)或胡椒(0)，三通道同步污染
        out.r[idx] = out.g[idx] = out.b[idx] = (rand() & 1) ? 255 : 0;
    }
    return out;
}
```

**设计要点**：

- **③ 固定随机种子**：可复现研究的必要条件，没有它量化验证就失了意义
- **④ 可能重复覆盖**：`rand() % total` 意味着同一像素可能被多次选中，实际噪声密度略低于设定值（15% 设定 → ~12.3% 实际，因为碰撞）。但验证指标使用运行时计算的**实际**噪声密度，所以不影响结论
- **⑤ 彩色椒盐噪声**：三通道同步污染，比通道独立噪声更具破坏性

### 2. 中值滤波核心

```cpp
Image medianFilter(const Image& src, int window) {
    Image out(src.w, src.h);                           // ① 输出图像
    int radius = window / 2;                           // ② 半径
    int area = window * window;                        // ③ 窗口面积
    std::vector<uint8_t> rv(area), gv(area), bv(area); // ④ 预分配缓冲区

    for (int y = 0; y < src.h; y++) {
        for (int x = 0; x < src.w; x++) {
            int n = 0;
            // ⑤ 收集窗口内所有像素
            for (int dy = -radius; dy <= radius; dy++) {
                int ny = y + dy;
                if ((unsigned)ny >= (unsigned)src.h) continue;  // ⑥ 边界跳过
                for (int dx = -radius; dx <= radius; dx++) {
                    int nx = x + dx;
                    if ((unsigned)nx >= (unsigned)src.w) continue;
                    int idx = ny * src.w + nx;
                    rv[n] = src.r[idx];   // ⑦ 三通道分别收集
                    gv[n] = src.g[idx];
                    bv[n] = src.b[idx];
                    n++;
                }
            }
            // ⑧ QuickSelect：只找中位数位置
            int mid = n / 2;
            std::nth_element(rv.begin(), rv.begin()+mid, rv.begin()+n);
            std::nth_element(gv.begin(), gv.begin()+mid, gv.begin()+n);
            std::nth_element(bv.begin(), bv.begin()+mid, gv.begin()+n);
            // ⑨ 输出中位数
            out.setPixel(x, y, rv[mid], gv[mid], bv[mid]);
        }
    }
    return out;
}
```

**逐行解析**：

**④ 预分配缓冲区**：在 262,144 次循环中，避免重复 `malloc/free`，节省数十万次内存分配

**⑥ 边界处理——"跳过"法**：三种常见边界处理策略的对比：
- 反射填充（Mirror）：镜像边界外像素，需严格像素计数时使用
- 零填充：简单但引入暗边伪影
- 跳过（Skip）：只收集窗口内实际存在的像素。对中值滤波是最佳选择——噪声在边界分布与内部相同，跳过缺失像素只是略微减小有效窗口大小，不影响中位数的统计性质

**⑧ `std::nth_element`**：
```cpp
std::nth_element(begin(), begin()+mid, begin()+n)
```
语义：将 `[begin(), begin()+n)` 部分重排，使 `begin()+mid` 位置是中位数（左边都 ≤ 它，右边都 ≥ 它），左右内部无序。平均复杂度 O(n)，比 `std::sort` 快 2-3 倍。

**一个常见陷阱**：`std::nth_element` 后，除了 `arr[mid]` 位置的元素外，不能信任任何其他位置的顺序！

### 3. PSNR 计算

```cpp
double computePSNR(const Image& a, const Image& b) {
    double mse = 0;
    for (int i = 0; i < a.w * a.h; i++) {
        double dr = (double)a.r[i] - b.r[i];
        double dg = (double)a.g[i] - b.g[i];
        double db = (double)a.b[i] - b.b[i];
        mse += (dr*dr + dg*dg + db*db) / 3.0;
    }
    mse /= (a.w * a.h);
    return 10.0 * log10(255.0 * 255.0 / mse);
}
```

公式：PSNR = 10 × log₁₀(MAX_I² / MSE)，其中 MAX_I = 255。

PSNR 解读：30-50 dB 高质量，20-30 dB 中等，<20 dB 低质量。噪声图像 PSNR = 13.07 dB，中值滤波后提升到 31.51 dB，**+18.4 dB 的提升**。

### 4. 边缘保持指数（EPI）

核心思想：对干净图像和滤波后图像分别计算 Sobel 梯度，然后计算两者梯度图之间的 Pearson 相关系数。越接近 1.0 说明边缘保持越好。

```
EPI = Σ(G_clean - μ_clean)(G_filt - μ_filt) / (σ_clean × σ_filt)
```

中值滤波 EPI = 0.952，高斯模糊 EPI = 0.701。95.2% vs 70.1% 的梯度相关性，中值滤波完胜。

### 5. 噪声移除率

```cpp
bool isNoise = (noisy.r[i] == 0 && clean.r[i] != 0) 
            || (noisy.r[i] == 255 && clean.r[i] != 255);
if (isNoise) {
    noiseCount++;
    // 滤波后的值是否比噪声值更接近原始值
    if (abs(filt.r[i] - clean.r[i]) < abs(noisy.r[i] - clean.r[i]))
        removed++;
}
```

关键定义：一个像素是噪声，当且仅当它在 noisy 中是 0/255 但在 clean 中不是。排除了原本就是黑色或白色的正常像素。

### 6. 均匀区域方差验证

在测试图像的白色方块区域（所有像素理论上都是 255），方差应该是 0。实际数据：

- 干净图像：方差 **0.0**（完美均匀）
- 噪声图像：方差 **4,110.9**（噪声波动极大）
- 中值滤波后：方差 **40.7**（恢复 99.0% 的均匀性）
- 高斯模糊后：方差 **257.3**（只恢复 93.7%）

中值滤波的残留方差只有高斯模糊的 **15.8%**——在保持均匀区域方面，中值滤波是高斯模糊的 **6.3 倍**。

## 踩坑实录

### 坑 #1：噪声密度参数选择不当

**症状**：最初 noiseDensity = 5%，中值滤波 PSNR 提升仅 3.2 dB，而且高斯模糊赢了（22.03 vs 20.96 dB）。与直觉矛盾——中值滤波专门为脉冲噪声设计，不应输给高斯模糊。

**错误假设**："噪声密度越高越能体现差异"是对的，但 5% 太低。512×512 上 5% 意味着 13,107 个噪声像素，大多数 3×3 窗口内只有 0-1 个噪声像素，中值和均值表现差不多。同时测试图像包含 1 像素宽的细线作为"边缘"，这些细线在 3×3 中值滤波下被抹掉，导致 EPI 只有 0.57。

**真实原因**：噪声密度太低 + 测试图像"边缘"太细。低密度下高斯和中值接近，而高斯对梯度背景的平滑反而帮它拿到了更高 PSNR。

**修复方案**：
1. 噪声密度提升到 **15%**（39,321 个噪声像素），3×3 窗口平均 1.35 个噪声像素
2. 重新设计测试图像，使用厚边缘（3 像素宽）代替细线
3. 使用大面积均匀色块作为方差测试基准

**经验教训**：选择合适的噪声密度**展示算法优势**和设计**能区分优劣的测试场景**同样重要。

### 坑 #2：缺少 `<cstdint>` 头文件

**症状**：`error: 'uint8_t' was not declared in this scope`

**错误假设**：`<cstdint>` 被 `<algorithm>` 或 `<vector>` 间接包含。在 GCC 13+ 可能成立，但在 GCC 11（当前环境）中不成立。

**真实原因**：`uint8_t` 是 C++11 `<cstdint>` 中的类型别名，不是基本类型。编译器较老时，标准库头文件不会自动包含它。

**修复方案**：显式添加 `#include <cstdint>`。不同版本 C++ 标准库在头文件依赖上有差异——始终显式包含所需类型的头文件，不要依赖间接包含。

### 坑 #3：PNG 文件大小误导

**症状**：PNG 转换后 original.png 只有 2.7KB，验证脚本要求 > 10KB。

**错误假设**：512×512×3 = 768KB 原始数据，PNG 压缩后应该足够大。

**真实原因**：测试图像使用大面积纯色块（四象限、白方块、黑方块）。PNG 的 DEFLATE 压缩对均匀区域极度高效——整块白色像素压缩后只有几十字节。

**修复方案**：
1. PPM 格式（未压缩）文件大小始终 786,447 字节（512×512×3 + 头部），是可靠的检测基准
2. 对 PNG 使用像素级验证（加载后检查尺寸和像素值范围），不依赖文件大小

### 坑 #4：`std::nth_element` 的语义陷阱

**症状**：以为 `nth_element` 后数组就有序了，访问其他位置出错。

**错误假设**：`std::nth_element` 类似于 `std::sort`，只是更快。

**真实原因**：`std::nth_element` 只保证：
- 第 n 个元素在正确位置
- 左边元素都 ≤ 它，右边元素都 ≥ 它
- **左右内部顺序未定义**

**修复方案**：只访问 `rv[mid]`、`gv[mid]`、`bv[mid]`，不依赖任何其他位置。

## 总结与延伸

### 技术局限性

1. **细纹理破坏**：中值滤波会破坏小于窗口大小的细节特征。对于包含丰富纹理的图像（如草皮、毛发），3×3 窗口就可能造成不可接受的细节损失
2. **转角与细线**：图像中的尖锐转角和细线在滤波后会被圆化或消失，这是中值滤波的固有缺陷
3. **高斯噪声不适用**：中值滤波专为脉冲噪声设计，对于高斯噪声（每个像素都有小幅随机扰动），高斯模糊是更好的选择
4. **计算复杂度**：朴素实现的 O(n × w² × log(w²)) 对于大窗口来说不轻量；虽然直方图增量更新可以将复杂度降到 O(n × w)，但在嵌入式系统上仍可能成为瓶颈
5. **不可分离性**：与高斯模糊不同，中值滤波不能拆分为两个 1D pass，限制了 GPU 上的并行化优化

### 可优化的方向

1. **自适应中值滤波（Adaptive Median Filter）**：根据局部噪声密度动态调整窗口大小。在低噪声区域用小窗口保持细节，高噪声区域用大窗口强力去噪。这是中值滤波最著名的改进版本。

2. **加权中值滤波（Weighted Median Filter）**：为窗口内不同位置的像素赋予不同权重（例如中心像素权重更高），然后取加权中位数。结合了中值滤波的稳健性和加权平均的空间信息。

3. **向量中值滤波（Vector Median Filter）**：对于彩色图像，当前的逐通道中值可能导致色调偏移。向量中值滤波同时考虑 R/G/B 三个分量，将窗口内各像素视为三维向量，取到其他向量距离之和最小的向量作为输出。

4. **基于直方图的快速实现**：使用直方图在 O(1) 时间内找到中位数（通过累积分布），每次滑动窗口只更新进入和离开的两列。对于 >5×5 的窗口，性能提升显著。

5. **GPU 并行化**：虽然中值滤波不可分离，但每个像素的计算是独立的。使用 compute shader 对每个像素并行处理，配合 shared memory 中的排序网络（如 Batcher's odd-even mergesort），可以在 GPU 上获得实时性能。

### 与系列其他文章的关联

- **Sobel 边缘检测（07-20）**：中值滤波可以作为边缘检测的**预处理步骤**——先去除椒盐噪声，再进行 Sobel 梯度计算，可以有效避免噪声像素产生虚假的梯度响应
- **直方图均衡化（07-15）**：直方图相关技术是中值滤波快速实现的数学基础——维护像素值直方图并通过累积分布找到中位数
- **双边滤波（07-01）**：双边滤波是另一个边缘保持滤波器，但它处理的是高斯噪声而非脉冲噪声。两者的设计哲学（"保持边缘的同时去噪"）一致但实现路径不同
