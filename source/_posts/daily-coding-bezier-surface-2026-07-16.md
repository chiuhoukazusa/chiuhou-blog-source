# 每日编程实践：贝塞尔曲面渲染器 (Bicubic Bezier Surface)

## ① 背景与动机

在计算机图形学中，表示光滑曲面的能力是几乎所有三维建模工具的基础。从 Maya 和 Blender 中的曲面建模，到汽车外形的工业设计，再到角色动画中的皮肤变形，光滑曲面无处不在。

**没有曲面表示时的痛点**：如果只用三角形网格，要表达一个光滑的曲面（比如汽车引擎盖）需要成千上万个三角形。这不仅浪费内存，也让编辑修改变得极其困难——移动一个顶点可能破坏光滑性，需要手动调整几十个相邻顶点来恢复形体的流畅感。

**贝塞尔曲面的优势**：仅需 16 个控制点（4×4 网格），就定义了任意精细程度的光滑曲面。这解决了三个核心问题：

1. **紧凑表示**：数据量极小，16 个点 vs 数千个三角面
2. **直觉编辑**：设计师拖动控制点即可直观地改变曲面形状——曲面会"跟随"控制点运动，但不一定穿过所有控制点
3. **数学保证光滑**：由于 Bernstein 多项式的 C^∞ 连续性，曲面在任意位置都是无限可微的

**工业界使用场景**：
- **汽车工业**：Pierre Bézier 本人在雷诺汽车公司工作时发明了贝塞尔曲线和曲面，用于汽车外形设计。至今 CATIA 等 CAD 软件仍以此为核心建模基元
- **动画与特效**：Bézier 曲面变形器（FFD，Free-Form Deformation）用于《玩具总动员》等影片中的角色动画，通过对控制晶格的操纵改变模型形状
- **字体设计**：TrueType 和 OpenType 字体中的曲线轮廓由二次贝塞尔曲线构成，PostScript 字体使用三次贝塞尔曲线

本文将从零实现一个双三次贝塞尔曲面渲染器，通过数学原理、代码实现、量化验证三步走，最终生成带 Phong 光照的曲面渲染图。

## ② 核心原理

### 2.1 从贝塞尔曲线到贝塞尔曲面

贝塞尔曲线由一组控制点和一个参数方程定义：

$$
\mathbf{P}(t) = \sum_{i=0}^{n} \mathbf{P}_i \cdot B_i^n(t), \quad t \in [0,1]
$$

其中 $B_i^n(t) = \binom{n}{i} t^i (1-t)^{n-i}$ 是 Bernstein 多项式。

**直觉理解**：可以把 $B_i^n(t)$ 理解为"第 $i$ 个控制点 $\mathbf{P}_i$ 在参数 $t$ 处的影响力"。当 $t=0$ 时，只有 $B_0^n(0)=1$（其他为 0），所以曲线起点恰好是 $\mathbf{P}_0$；当 $t=1$ 时同理，终点恰好是 $\mathbf{P}_n$。这是**端点插值性质**——贝塞尔曲线穿过首尾控制点，但不一定穿过中间控制点。

**贝塞尔曲面是曲线的张量积扩展**。给定一个 $4\times4$ 的控制网格 $\{\mathbf{P}_{ij}\}$，曲面点由两个方向的参数 $(u,v) \in [0,1]^2$ 决定：

$$
\mathbf{S}(u,v) = \sum_{i=0}^{3}\sum_{j=0}^{3} \mathbf{P}_{ij} \cdot B_i^3(u) \cdot B_j^3(v)
$$

**直觉解释**：
- 固定 $v$，$\mathbf{S}(u, v_0)$ 是一条贝塞尔曲线（控制点是 $\sum_j \mathbf{P}_{ij} B_j^3(v_0)$）
- 固定 $u$，$\mathbf{S}(u_0, v)$ 是另一条贝塞尔曲线
- 整个曲面是"双向曲线"的张量积——在 $u$ 和 $v$ 两个方向同时插值

这个公式的优美之处在于：它把二维曲面问题分解为两个一维贝塞尔曲线问题的乘积，极大地简化了计算。对于任意 $(u,v)$，我们只需要计算 16 个 Bernstein 基函数的值，做 16 次加权平均即可得到空间中的点。

---

### 2.2 Bernstein 多项式

三次 Bernstein 基函数是：

$$
\begin{aligned}
B_0^3(t) &= (1-t)^3 \\
B_1^3(t) &= 3t(1-t)^2 \\
B_2^3(t) &= 3t^2(1-t) \\
B_3^3(t) &= t^3
\end{aligned}
$$

这些基函数具有关键的**单位分割**性质：对所有 $t$，$\sum_{i=0}^{3} B_i^3(t) = 1$。这意味着曲面上的任何点都是控制点的凸组合，曲面始终落在控制网格的凸包内。这是为什么贝塞尔曲面"行为可预测"的数学保证——设计师可以确信曲面不会突然飞到控制网格外面去。

**一般形式**：
$$B_i^n(t) = \binom{n}{i} t^i (1-t)^{n-i}$$

其中二项式系数 $\binom{n}{i} = \frac{n!}{i!(n-i)!}$ 决定每个控制点的最大影响力。

下面是一张各基函数在 $t\in[0,1]$ 上的形状描述：

- $B_0^3(t)$：在 $t=0$ 处为 1，单调递减到 0——控制点 $\mathbf{P}_0$ 在曲线的起点处影响最大
- $B_1^3(t)$：在 $t=0$ 处为 0，先增后减到 0——$\mathbf{P}_1$ 在 $t \approx 1/3$ 处影响最大
- $B_2^3(t)$：形状类似 $B_1^3$ 但对称到 $t \approx 2/3$——$\mathbf{P}_2$ 在靠右半段起作用
- $B_3^3(t)$：在 $t=0$ 处为 0，单调递增到 1——$\mathbf{P}_3$ 在曲线末尾区域主导

---

### 2.3 曲面法线计算

光照计算需要曲面在每个点上的法线。对于参数曲面，法线 = 两个切向量的叉积：

$$
\mathbf{N}(u,v) = \frac{\partial \mathbf{S}}{\partial u} \times \frac{\partial \mathbf{S}}{\partial v}
$$

**为什么是叉积？** 切向量 $\partial \mathbf{S}/\partial u$ 是曲面上"沿 $u$ 方向移动"的瞬时方向，$\partial \mathbf{S}/\partial v$ 同理。这两个切向量张成了曲面在该点的切平面，而法线垂直于该切平面。叉积恰好产生了这样一个垂直于两个切向量的向量。

**叉积的几何含义**：$\mathbf{a} \times \mathbf{b}$ 的长度等于以 $\mathbf{a},\mathbf{b}$ 为边的平行四边形的面积。这个"面积"越大，说明曲面在该点越"陡峭"（两个切向量张得越开）。归一化后的单位法线方向则决定了光照中漫反射和高光的方向。

偏导数的计算通过 Bernstein 基函数的导数完成。三次 Bernstein 基函数的导数为：

$$
\begin{aligned}
\frac{dB_0^3}{dt} &= -3(1-t)^2 \\
\frac{dB_1^3}{dt} &= 3(1-t)(1-3t) \\
\frac{dB_2^3}{dt} &= 3t(2-3t) \\
\frac{dB_3^3}{dt} &= 3t^2
\end{aligned}
$$

**为什么偏导数如此重要？** 因为光照方程中的漫反射项 $\max(0, \mathbf{N}\cdot\mathbf{L})$ 和镜面反射项都依赖于法线方向。如果法线计算错误（例如用三角形面的法线近似），曲面会显得多面体化，失去了光滑渲染的优势。

利用链式法则，曲面对 $u$ 的偏导数是：

$$
\frac{\partial \mathbf{S}}{\partial u} = \sum_{i=0}^{3}\sum_{j=0}^{3} \mathbf{P}_{ij} \cdot \frac{dB_i^3(u)}{du} \cdot B_j^3(v)
$$

对 $v$ 的偏导数类似，将导数放到 $B_j^3(v)$ 上即可。

**数值稳定性注意事项**：当切向量平行时（如在参数域的退化点），叉积为零向量，无法得到有意义的法线。在实际应用中，贝塞尔曲面在参数域的角点处是双线性插值产生的退化点，但由于我们选取的控制网格在合理范围内，四个角点处切向量不会平行，能产生良好定义的曲面法线。

### 2.4 与其他曲面表示的比较

| 方法 | 优点 | 缺点 |
|------|------|------|
| **贝塞尔曲面** | 简洁优雅，端点插值保证，设计师友好 | 移动一个控制点影响整个曲面（全局性） |
| **B-样条曲面** | 局部控制，更高的灵活性 | 更复杂的结点向量（knot vector）配置 |
| **NURBS 曲面** | 最通用，能精确表示圆锥曲线（圆/椭圆等） | 权重调整不直观 |
| **Catmull-Clark 细分** | 支持任意拓扑，电影级曲面 | 需要迭代细分，实时渲染性能差 |
| **三角形网格** | GPU 原生加速，工具链完善 | 大量数据，编辑不平滑 |

选择贝塞尔曲面是因为它是所有参数曲面的基础，理解它就理解了 B-样条和 NURBS 的出发点。B-样条本质上是把多个贝塞尔曲线段拼接起来并保证连续性。

## ③ 实现架构

### 3.1 数据流概览

```
控制网格 (4×4 Vec3)
    │
    ├─→ 参数域采样 (50×50 tessellation)
    │     ├─→ evalBezier(u,v) → 3D 位置
    │     ├─→ evaldU(u,v) × evaldV(u,v) → 法线
    │     └─→ 透视投影 → 屏幕坐标
    │
    ├─→ 三角形光栅化（逐像素）
    │     ├─→ Z-Buffer 深度测试
    │     ├─→ Phong 着色（漫反射 + 镜面反射 + 环境光）
    │     └─→ 帧缓冲颜色写入
    │
    └─→ PPM 输出 + 量化验证
```

### 3.2 关键数据结构

```cpp
// 3D 向量：用 float 三元组表示
struct Vec3 { float x, y, z; };

// 颜色：字节三元组（范围 0-255）
struct Color { unsigned char r, g, b; };

// Z-Buffer：1D float 数组，大小 WIDTH*HEIGHT
// 帧缓冲：1D Color 数组，大小 WIDTH*HEIGHT
std::vector<float> zbuffer(WIDTH * HEIGHT, 1e10f);
std::vector<Color> framebuffer(WIDTH * HEIGHT, ...);
```

为什么选择这些数据结构？

- **Z-Buffer 用 float** 而非 short 或 int：虽然精度要求不高，但 float 避免了定点数转换的复杂性，且深度比较直接
- **Color 用 unsigned char**：PPM P6 格式本身就是 RGB 24-bit，字节对齐，直接 memcpy 即可写入
- **1D 数组拍平**：WIDTH*HEIGHT 的 1D 布局比 2D 数组更快（缓存友好），且删除越界访问的风险

### 3.3 曲面细分策略（Tessellation）

曲面在参数空间 $(u,v)$ 上进行均匀采样：`TESS=50` 表示 $50\times50$ 个四边形面片。为什么是 50？

- 太少（如 10×10）：四边形过大，投影后的屏幕覆盖率不够，产生孔洞
- 太多（如 200×200）：计算量增加 16 倍，但视觉上 50×50 已经足够，因为每个四边形覆盖 20×15 像素左右
- **性能分析**：50×50 = 2500 个四边形，每个内循环跑约 300 次像素填充（在投影面积内的包围盒），总计约 750k 次像素操作，对于软光栅化来说足够

**为什么不自适应细分？** 自适应细分（在曲率大的地方多采样，平坦处少采样）可以进一步优化，但增加了裂缝（cracking）的风险——相邻面片细分级别不同时会产生 T 形顶点。对这么小的控制网格，均匀细分足够好。

## ④ 关键代码解析

### 4.1 Bernstein 多项式计算

```cpp
float bernstein(int i, int n, float t) {
    float coeff = 1.0f;
    for (int j = 1; j <= i; ++j) 
        coeff = coeff * (n - j + 1) / j;
    return coeff * std::pow(t, i) * std::pow(1.0f - t, n - i);
}
```

**为什么用循环计算组合数而不是预计算？** 虽然 $\binom{3}{i}$ 可以手算（1, 3, 3, 1），但循环写法保持了通用性——如果将来改用 5 次贝塞尔曲面（6 个控制点），只需改一行参数。循环使用**乘法递减法**避免浮点阶乘溢出：

$$\binom{3}{2} = \frac{3 \times 2}{2 \times 1} = 3$$

### 4.2 曲面求值

```cpp
Vec3 evalBezier(const Vec3 ctrl[4][4], float u, float v) {
    Vec3 result;
    for (int i = 0; i < 4; ++i)
        for (int j = 0; j < 4; ++j)
            result = result + ctrl[i][j] 
                     * (bernstein(i, 3, u) * bernstein(j, 3, v));
    return result;
}
```

这是张量积曲面定义公式的直接翻译。$B_i^3(u) \times B_j^3(v)$ 给出了控制点 $\mathbf{P}_{ij}$ 在参数 $(u,v)$ 处的权重。

**张量积展开的计算量分析**：
对于每个 $(u,v)$ 采样点，`evalBezier` 做了 16 次 Bernstein 基函数求值和 16 次向量加权。其中每次 `bernstein` 涉及 1 次 `std::pow` 和 1 次循环（4 次迭代最坏情况）。总计每个采样点约 64 次乘法和 16 次 `std::pow` 调用。对于 50×50 的细分，共 2500 个采样点，$\approx 160k$ 次乘法和 $40k$ 次 `std::pow`——在 CPU 上完全可实时完成（< 10ms）。

**与 de Casteljau 算法的对比**：
de Casteljau 算法通过递归线性插值求贝塞尔曲线上的点：

```
P_i^0 = P_i                           (初始控制点)
P_i^k = (1-t) * P_i^{k-1} + t * P_{i+1}^{k-1}   (k 次迭代)
P_0^n = 曲线上的点
```

de Casteljau 对于单条曲线需要 $O(n^2)$ 次线性插值（n=3 时为 6 次），而 Bernstein 展开法需要 4 次 $t^i (1-t)^{n-i}$ 求值 + 4 次加法。两种方法在数学上等价，但 de Casteljau 更**数值稳定**：它只用加法乘法，避免了 $t=0$ 或 $t=1$ 处 $0^0$ 的数值问题。在真正的工业代码中，de Casteljau 是首选的求值方法。本实现使用 Bernstein 展开是因为它更直接映射到数学公式，便于理解。

**容易写错的地方**：
- 混淆 `i` 和 `j` 分别对应 `u` 还是 `v`——这里 `i` 索引对应 $u$ 方向（列），`j` 对应 $v$ 方向（行），与控制网格 `ctrl[i][j]` 一致
- 忘记录入所有 16 个控制点——编译器不会报错，但曲面会变形。务必确认控制网格的形状是你想要的
- 忘了归一化法线——光照模型假设法线是单位向量，`cross()` 后的向量长度等于两切向量张成的平行四边形面积，不一定是 1

### 4.3 偏导数与法线

```cpp
// Bernstein 基函数的导数
float dbernstein(int i, float t) {
    if (i == 0) return -3.0f * std::pow(1.0f - t, 2);
    if (i == 1) return 3.0f * (1.0f - t) * (1.0f - 3.0f * t);
    if (i == 2) return 3.0f * t * (2.0f - 3.0f * t);
    return 3.0f * t * t;  // i == 3
}

// 对 u 的偏导数
Vec3 evaldU(const Vec3 ctrl[4][4], float u, float v) {
    Vec3 result;
    for (int i = 0; i < 4; ++i)
        for (int j = 0; j < 4; ++j)
            result = result + ctrl[i][j] 
                     * (dbernstein(i, u) * bernstein(j, 3, v));
    return result;
}

// 法线 = 两个切向量的叉积的归一化
Vec3 evalNormal(const Vec3 ctrl[4][4], float u, float v) {
    Vec3 du = evaldU(ctrl, u, v);
    Vec3 dv = evaldV(ctrl, u, v);
    return du.cross(dv).normalize();
}
```

`dbernstein` 展开为显式公式而非调用通用导数，原因有三：
1. **性能**：每次曲面求值调 16 次 `bernstein`，展开可以避免分支和循环
2. **稳定性**：避免在 $t=0$ 或 $t=1$ 处数值微分的不稳定
3. **正确性**：确保导数的闭式表达一致于定义

### 4.4 透视投影

```cpp
auto proj = [&](const Vec3& p) -> std::pair<int,int> {
    float px = p.x / (camZ - p.z) * scale + cx;
    float py = -p.y / (camZ - p.z) * scale + cy;
    return {(int)(px + 0.5f), (int)(py + 0.5f)};
};
```

关键的 `-p.y` 是因为屏幕坐标的 Y 轴向下（PPM 左上角为原点），而世界空间的 Y 轴向上。这个 `-` 号是最容易被遗忘的细节。*（历史教训：2/20 的坐标系 Bug 就是因为忘了这个翻转）*

投影使用单点透视（类似 pinhole camera 模型）。`scale` 参数计算自 `fov`：

```cpp
float fov = 60.0f * M_PI / 180.0f;
float scale = HEIGHT * 0.5f / std::tan(fov * 0.5f);
```

$ \tan(30^\circ) = 1/\sqrt{3} \approx 0.577 $，取 $scale = 384 / 0.577 \approx 665$。

### 4.5 光栅化循环与 Phong 着色

```cpp
for (int iu = 0; iu < TESS; ++iu) {
    for (int iv = 0; iv < TESS; ++iv) {
        // 4 个角点 + 4 个法线
        Vec3 p00 = evalBezier(ctrl, u0, v0);  // ... etc
        Vec3 n00 = evalNormal(ctrl, u0, v0);  // ... etc
        
        // 投影到屏幕
        // 计算包围盒
        for (int y = minY; y <= maxY; ++y) {
            for (int x = minX; x <= maxX; ++x) {
                // Z-Buffer 深度测试
                float depth = camZ - pos.z;
                if (depth >= zbuffer[idx]) continue;
                
                // Phong 光照
                Vec3 n = bilinear_interpolate(n00,n10,n01,n11,ax,ay);
                float NdotL = clamp(n.dot(lightDir), 0, 1);
                float spec = pow(clamp(n.dot(halfVector), 0, 1), shininess);
                Vec3 col = ambient + diffuse * NdotL + specular * spec;
                
                zbuffer[idx] = depth;
                framebuffer[idx] = Color(col);
            }
        }
    }
}
```

**Phong 光照模型的三项**：

1. **环境光** (`ambient * 0.3`)：模拟多次散射的恒定光，防止背面全黑。乘 0.3 是因为环境光只是近似，不宜过强
2. **漫反射** (`diffuse * NdotL`)：Lambertian 反射，$\max(\mathbf{N}\cdot\mathbf{L}, 0)$ 表示光照随法线与光源夹角余弦衰减
3. **镜面反射** (`specular * spec^32`)：Blinn-Phong 近似，用半角向量 $\mathbf{H}=(\mathbf{L}+\mathbf{V})/\|\mathbf{L}+\mathbf{V}\|$ 代替反射向量计算，性能更好且视觉差别很小

**为什么用 Z-Buffer 而不是画家算法？** 贝塞尔曲面本身是没有自遮挡的简单曲面，但多个四边形面片会在屏幕空间重叠（尤其在曲面弯折处）。Z-Buffer 以像素级精度解决了遮挡问题，不需要为四边形排深度序。

## ⑤ 踩坑实录

### 坑 1：法线方向反转

**症状**：渲染的曲面一侧亮一侧暗，但暗的表面应该是朝光源的。

**错误假设**：以为 `evaldU × evaldV` 自动产生正确的向外法线。

**真实原因**：叉积的方向取决于 $(u,v)$ 参数化方向。对于我这个控制网格，`du × dv` 在某些区域指向上表面，在某些区域指向下表面，因为参数化的方向和我直观认知的"外表面"方向不一致。

**修复方式**：对每个四边形，比较法线 $\mathbf{N}$ 和视角向量 $\mathbf{V}$ 的点积。当 $\mathbf{N} \cdot \mathbf{V} < 0$（法线背向摄像机），手动翻转法线方向。但在我的实现中，经过仔细设置控制网格后，所有可见面的法线都正确朝外，不需要翻转。

### 坑 2：包围盒内所有像素都被着色

**症状**：四边形之间的间隙和不平滑的渲染边缘。

**错误假设**：包围盒内的所有像素都属于当前四边形。

**真实原因**：投影后的四边形不一定是凸的（在特定视角下），包围盒覆盖了本应属于相邻四边形甚至背景的像素。

**修复方式**：Z-Buffer 深度测试解决了这个问题——只有距离相机最近的像素被写入最终帧缓冲。如果两个四边形重叠在同一像素上，较近的一个会赢得 Z-Buffer 竞争。

### 坑 3：Color 构造函数的歧义调用

**症状**：编译失败，报错 `call of overloaded 'Color(<brace-enclosed initializer list>)' is ambiguous`。

**错误假设**：`Color{30, 50, 80}` 会自动推断为 `Color(unsigned char, unsigned char, unsigned char)`。

**真实原因**：C++ 中 `{30, 50, 80}` 既可以匹配 `Color(unsigned char, unsigned char, unsigned char)` 也可以匹配 `Color(float, float, float)`——因为整数字面量同时可以隐式转换到 `unsigned char` 和 `float`。

**修复方式**：
1. 方案 A：用 `Color(30, 50, 80)` 替代 `Color{30, 50, 80}`，圆括号初始化优先匹配精确类型
2. 方案 B：删除 `Color(float, float, float)` 构造，改用静态工厂方法 `Color::fromFloat()`——这是我最终的选择，它在语义上更清晰

### 坑 4：单元测试被跳过

**症状**：代码编译成功、运行成功、输出 PPM 文件，但量化验证输出显示 P(0,1) 和 P(1,0) 的期望值写反了。

**错误假设**：以为肉眼一看就知道"渲染正确了"。

**真实原因**：没有认真分析每个测试用例的输出。实际上是期望值写错了（我把 u/v 对应的角点搞反了），实际计算完全正确。**这就是为什么量化验证不能省略**——如果靠眼睛看，"看起来没错" 和"精确无误"之间的差距可能永远发现不了。

### 6.3 端点插值性质

贝塞尔曲面的理论特性：角点恰好穿越对应的控制顶点。这是由 Bernstein 基函数在端点的特殊性质保证的：

$$
B_0^3(0)=1, B_1^3(0)=B_2^3(0)=B_3^3(0)=0
$$
$$
B_3^3(1)=1, B_0^3(1)=B_1^3(1)=B_2^3(1)=0
$$

在 $(u,v)=(0,0)$ 处，只有 $B_0^3(0) \cdot B_0^3(0) = 1$，其他 15 个基函数的乘积都是 0，所以 $\mathbf{S}(0,0) = \mathbf{P}_{00}$。同理可证其他三个角点的插值性质。验证结果：

- P(0,0) = (-3.000, -3.000, 2.000) = 控制点 [0][0] ✅
- P(1,1) = (3.000, 3.000, 2.000) = 控制点 [3][3] ✅
- P(0,1) = (-3.000, 3.000, 2.000) = 控制点 [0][3] ✅
- P(1,0) = (3.000, -3.000, 2.000) = 控制点 [3][0] ✅

**平均角点误差：0.000000**（浮点精度完全一致），证实 Bernstein 基函数的端点插值性质实现正确。

### 6.4 鞍面曲率分析

我设计的控制网格将中心四个控制点（索引 [1][1]、[1][2]、[2][1]、[2][2]）的 Z 坐标压低到 -2.5，而角点和边缘在 +2.0。这创造了一个**双曲鞍面**（hyperbolic paraboloid）：

- 角点平均 Z：2.000
- 中心点 Z：-0.531（在控制点凹陷处上方）
- Z 相差 2.531，具有明显的鞍形曲率

这是贝塞尔曲面的全局控制特性体现：中心区虽然被压低到 -2.5，但因为周围控制点的拉拽（拉力通过 Bernstein 权重传播），中心实际 Z 值"仅"降到 -0.531。这说明设计师压低中心控制点时，曲面其他部分也会受到委婉的影响。

## ⑦ 总结与延伸

### 技术局限性

1. **全局控制**：移动一个控制点会改变整个曲面。如果你想"只在某个位置抬高曲面而不影响其他区域"，基本做不到。这是贝塞尔方法的天生缺陷，需要 B-样条或 NURBS 解决
2. **矩形拓扑限制**：贝塞尔曲面只能是矩形参数域。对于球面、环面等非矩形拓扑，需要多个贝塞尔面片拼接或使用 NURBS
3. **控制点数固定**：三次贝塞尔面固定 4×4 网格。如果想增加细节控制，只能换更高级的多项式或增加面片
4. **透视投影局限**：本实现使用最简透视投影。如果要渲染带有复杂光源、纹理映射、阴影的场景，需要完整的渲染管线

### 可优化方向

1. **自适应细分**：根据曲面曲率动态调整 tessellation 密度。在接近平坦处降低分辨率，在高曲率处增加，节省计算
2. **de Casteljau 求值**：比 Bernstein 多项式展开法更数值稳定，且适合递归细分（二分 recursively splitting）
3. **法线贴图增强**：在贝塞尔曲面上叠加法线贴图，在不增加 tessellation 的前提下大幅度增加视觉细节
4. **GPU 曲面细分着色器**：现代 GPU 的 Tessellation Shader 可以直接在硬件上对参数曲面进行自适应细分

### 与本系列的关系

- **02-22 Bezier 曲线**：本文曲面的 1D 基础——理解 Bezier 曲线才能理解拉格朗日曲面
- **05-26 Catmull-Clark 细分曲面**：另一种曲面表示方法，支持任意拓扑——在网格类建模中更通用
- **06-17 Delaunay 三角剖分**：曲线与三角剖分构成离散计算几何和光滑 CAGD 的两大分支
