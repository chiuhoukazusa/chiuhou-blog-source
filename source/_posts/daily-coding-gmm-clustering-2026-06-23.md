## ① 背景与动机

聚类(Clustering)是无监督学习中最基础也最广泛使用的技术之一。给定一堆没有标签的数据点，我们希望自动发现其中的"自然分组"——把相似的点归到一起，把不同的点分开。

最简单的聚类算法是 K-Means，它的核心思想是：每个点只属于距离最近的聚类中心。这个"硬指派"(hard assignment)虽然简单高效，但有三个致命缺陷：

**缺陷一：不能表达不确定性。** K-Means 说"这个点100%属于类别A"，但现实中的数据和人的认知往往是模棱两可的。比如一个位于两个聚类边界上的点，它可能30%属于A、70%属于B——这才是更合理的描述。

**缺陷二：假设所有聚类都是球形。** K-Means 用欧氏距离来判断归属，隐含的假设是每个聚类在各方向上的"散度"是一样的（各向同性）。现实中的聚类可能是长条形的、倾斜的、不同大小的——K-Means 对此无能为力。

**缺陷三：没有概率解释。** K-Means 不告诉你"这个聚类有多大可能性出现"——它只告诉你每个点属于哪个聚类，但没有告诉你整个模型的"不确定性"或"置信度"。

高斯混合模型(Gaussian Mixture Model, GMM)就是为了解决这些缺陷而生的。它假设数据是由 K 个高斯分布（正态分布）按一定权重混合生成的，每个高斯分布有自己的均值（中心）和协方差矩阵（决定聚类的形状、方向和大小）。

GMM 的应用场景非常广泛：
- **图像分割**：GrabCut 算法的核心就是用 GMM 建模前景和背景的颜色分布
- **异常检测**：GMM 可以给出每个样本的生成概率，概率极低的点就是异常
- **语音识别**：GMM 是传统声学模型 HMM-GMM 的核心——每个音素由一个高斯混合模型建模
- **目标跟踪**：在线 GMM 用于建模动态背景（MOG2 背景减除）
- **市场细分**：客户行为数据聚类，每个客户以概率形式属于多个细分市场

今天我们从头实现一个完整的 GMM，包含 EM 算法的完整推导、BIC 模型选择的自动化（不必预先知道 K），以及量化的验证指标——而不是"看着图觉得对"。

## ② 核心原理

### 2.1 高斯混合模型的定义

一个具有 K 个分量的高斯混合模型定义为：

\[
p(x) = \sum_{k=1}^{K} \pi_k \cdot \mathcal{N}(x | \mu_k, \Sigma_k)
\]

其中：
- \(\pi_k\) 是第 k 个分量的**混合系数**(mixing coefficient)，满足 \(\sum_{k=1}^K \pi_k = 1\) 且 \(\pi_k \geq 0\)。直观上，\(\pi_k\) 表示"随机抽一个数据点，它来自第 k 个分布的概率"。
- \(\mu_k\) 是第 k 个高斯分量的**均值向量**，即聚类的中心位置。
- \(\Sigma_k\) 是第 k 个高斯分量的**协方差矩阵**，它决定了聚类在空间中的形状、旋转方向和大小——这是 GMM 相比 K-Means 最关键的升级。
- \(\mathcal{N}(x|\mu_k, \Sigma_k)\) 是多元高斯分布的**概率密度函数**(PDF)：

\[
\mathcal{N}(x|\mu, \Sigma) = \frac{1}{(2\pi)^{D/2} |\Sigma|^{1/2}} \exp\left(-\frac{1}{2}(x-\mu)^T \Sigma^{-1} (x-\mu)\right)
\]

这个公式的直觉含义是：\((x-\mu)^T\Sigma^{-1}(x-\mu)\) 计算的是点 x 到中心 μ 的**马氏距离**(Mahalanobis distance)——协方差矩阵在这里的作用就像是一种"度量变形"，椭圆的聚类内部距离自然比球形的小。

### 2.2 为什么需要 EM 算法？

如果我们知道每个数据点属于哪个高斯分量（即知道"隐变量"），那么参数估计就很简单：
- μ 就是属于该分量的所有点的均值
- Σ 就是这些点的协方差矩阵
- π 就是该分量的点数除以总点数

但问题恰恰是：**我们不知道每个点属于哪个分量**。这就是"不完全数据"(incomplete data)问题。

EM 算法(Expectation-Maximization)正是解决不完全数据参数估计的标准方法：
- **E-step**：基于当前参数估计，计算每个点属于每个分量的后验概率（"猜测"隐变量）
- **M-step**：基于这些后验概率，重新估计参数（让当前"猜测"下的似然最大）

这个过程交替进行，每次迭代都保证对数似然**非递减**——这是 EM 算法最核心的理论保证。

### 2.3 E-step：计算后验概率（责任度）

定义隐变量 \(z_{ik}\) 表示数据点 \(x_i\) 是否来自第 k 个分量（是则为 1，否则为 0）。E-step 计算的是给定当前参数 \(\theta^{(t)}\) 和数据点 \(x_i\) 时，\(z_{ik}=1\) 的后验概率：

\[
\gamma_{ik} = P(z_{ik}=1 | x_i, \theta^{(t)}) = \frac{\pi_k \cdot \mathcal{N}(x_i|\mu_k, \Sigma_k)}{\sum_{j=1}^K \pi_j \cdot \mathcal{N}(x_i|\mu_j, \Sigma_j)}
\]

\(\gamma_{ik}\) 被称为**责任度**(responsibility)——第 k 个分量对解释数据点 \(x_i\) 承担了多少"责任"。

**直观理解**：分子是"如果 x_i 真的来自分量 k，它被观测到的概率乘以分量 k 的先验概率"，分母是所有分量的这个量的和。所以如果分量 k 的 PDF 值远大于其他分量，\(\gamma_{ik}\) 就接近 1——分量 k 对 x_i 的解释能力最强。

注意这里与 K-Means 的本质不同：K-Means 的"归属"是 0 或 1（硬指派），而 GMM 的 \(\gamma_{ik}\) 是一个 0 到 1 之间的实数（软指派）。这就是"软聚类"(soft clustering)的含义。

### 2.4 M-step：最大化期望对数似然

M-step 利用 E-step 计算的责任度来更新参数。核心思路是**最大化完整数据对数似然在当前后验分布下的期望**（这被称为 Q 函数，EM 的"E"指的就是这个期望）：

\[
Q(\theta|\theta^{(t)}) = \sum_{i=1}^N \sum_{k=1}^K \gamma_{ik} \left[ \log \pi_k + \log \mathcal{N}(x_i|\mu_k, \Sigma_k) \right]
\]

对 \(\mu_k\)、\(\Sigma_k\) 和 \(\pi_k\) 分别求导并设为零，得到更新公式：

**均值更新**：
\[
\mu_k^{(t+1)} = \frac{\sum_{i=1}^N \gamma_{ik} \cdot x_i}{\sum_{i=1}^N \gamma_{ik}} = \frac{1}{N_k} \sum_{i=1}^N \gamma_{ik} \cdot x_i
\]

其中 \(N_k = \sum_{i=1}^N \gamma_{ik}\) 是分配给第 k 个分量的"有效点数"。

**直觉**：新的均值是所有数据点的加权平均，权重是各点对分量 k 的责任度。一个点越"像"分量 k，它在计算分量 k 的新均值时就越重要。

**协方差更新**：
\[
\Sigma_k^{(t+1)} = \frac{\sum_{i=1}^N \gamma_{ik} \cdot (x_i - \mu_k^{(t+1)})(x_i - \mu_k^{(t+1)})^T}{\sum_{i=1}^N \gamma_{ik}}
\]

**直觉**：这是责任度加权的外积平均——如果两个分量有重叠，边界上的点会对两个分量都贡献协方差（但权重较低），这自然产生了"软边界"。

**权重更新**：
\[
\pi_k^{(t+1)} = \frac{N_k}{N} = \frac{\sum_{i=1}^N \gamma_{ik}}{N}
\]

**直觉**：权重就是"有多少点的解释主要归因于该分量"除以总数。

### 2.5 收敛性：监考对数似然

EM 算法有一个漂亮的理论性质：**每次迭代后，观测数据的对数似然(l)不会下降**：

\[
\log p(X|\theta^{(t+1)}) \geq \log p(X|\theta^{(t)})
\]

因此，我们可以监控对数似然的变化来判断收敛：

\[
\log p(X|\theta) = \sum_{i=1}^N \log\left(\sum_{k=1}^K \pi_k \cdot \mathcal{N}(x_i|\mu_k, \Sigma_k)\right)
\]

注意这里有两个求和：外层对所有数据点，内层对所有分量。对每个数据点，先计算它在所有分量上的加权 PDF 和，再取对数。

当两次迭代的对数似然差小于阈值（如 \(10^{-5}\)）时，我们认为 EM 已收敛。这个收敛检测**必须量化**——不能"看着差不多的"就觉得收敛了。

### 2.6 BIC：自动选择最佳聚类数 K

K-Means 和原始 GMM 都需要**预先指定聚类数 K**。但现实中我们往往不知道数据有几类。解决方案是**贝叶斯信息准则**(Bayesian Information Criterion, BIC)：

\[
\text{BIC} = -2 \cdot \log L + p \cdot \log N
\]

其中：
- \(\log L\) 是最优的对数似然值
- \(p\) 是模型参数的总数（对 D 维空间的 GMM 有 K 个分量：\(p = K \cdot (D + D(D+1)/2 + 1) - 1\)。在 2D 下，每个分量有 1 个权重、2 个均值参数、3 个协方差参数，所以 \(p = 6K - 1\)（权重之和为 1，去掉一个自由度））
- \(N\) 是数据点总数

**BIC 的直觉**：第一项 \(-2\log L\) 衡量模型对数据的拟合程度（越小越好），第二项 \(p\log N\) 是对模型复杂度的惩罚（参数越多，惩罚越重）。BIC 自动在"拟合好"和"模型简单"之间找平衡。

实践中，我们对 K=1,2,3,...,K_max 分别训练 GMM，选 BIC 最小的那个 K。这正是 Occam's Razor（奥卡姆剃刀）的数学形式化：如果较简单的模型已经足够解释数据，就不要用更复杂的。

## ③ 实现架构

### 3.1 整体数据流

```
输入: 二维点集 {x_i} (N个点)
         ↓
多K训练: K = 2, 3, 4, 5 分别训练一个 GMM
         ↓
BIC 比选: 选择 BIC 最小的 K 作为最优模型
         ↓
聚类预测: 用最优 K 的 GMM 对每个点计算最大后验概率类别
         ↓
PPM 可视化: 二维平面上的点着色 + 95% 置信椭圆 + 聚类中心
         ↓
量化验证: BIC 比选正确性、对数似然收敛性、聚类均衡性、均值恢复精度
```

### 3.2 数据结构设计

```cpp
// 二维向量（用于 2D 数据的均值和对数似然梯度）
struct Vec2 { double x, y; };

// 2×2 对称矩阵（协方差矩阵）
struct Mat2 { double a, b, c, d; };

// 单个高斯分量
struct Gaussian {
    Vec2 mean;        // 均值向量
    Mat2 cov;         // 协方差矩阵
    Mat2 cov_inv;     // 缓存：协方差逆矩阵（加速 PDF 计算）
    double det_cov;   // 缓存：协方差行列式
    double weight;    // 混合系数 π_k
    
    void update_cache();  // 每次 M-step 后更新 cov_inv 和 det_cov
    double pdf(Vec2 x) const; // 概率密度函数
};
```

**设计说明**：
- **缓存逆矩阵和行列式**：每次 PDF 计算都需要用到 \(\Sigma^{-1}\) 和 \(\det(\Sigma)\)，在每次 M-step 后缓存这两个值可以大幅减少重复计算。
- **正则化**：协方差矩阵的行列式可能趋近于 0（导致数值不稳定），为此在 M-step 中对协方差对角线加上 \(10^{-6}\) 的正则化项。
- **权重归一化**：M-step 中按 \(\pi_k = N_k / \sum_j N_j\) 自然保证权重之和为 1。

### 3.3 CPU/数学侧职责划分

本实现完全运行在 CPU 上（数据量 600 点，不需要 GPU）：
- **GMM 类**：管理 K 个高斯分量，实现 EM 训练的完整流程
- **DataGenerator 类**：从指定的 \(\{\mu_k, \Sigma_k, \pi_k\}\) 参数生成合成数据，用于验证算法正确性
- **可视化**：纯 C++ 生成 PPM 图像，不依赖任何外部库
- **量化验证**：BIC 比较、对数似然跟踪、均值恢复误差——全部由标准输出日志驱动

## ④ 关键代码解析

### 4.1 E-step：责任度计算

这是 GMM 最核心的计算——每个数据点对每个分量的后验概率。代码必须同时做到正确和数值稳定：

```cpp
std::vector<std::vector<double>> e_step(const std::vector<Vec2>& data) {
    int N = data.size();
    // 责任度矩阵: N 行 × K 列，resp[i][k] = γ_{ik}
    std::vector<std::vector<double>> resp(N, std::vector<double>(K, 0.0));
    
    for (int i = 0; i < N; i++) {
        double total = 0.0;
        // Step 1: 计算未归一化的后验（分子）
        for (int k = 0; k < K; k++) {
            resp[i][k] = components[k].weight * components[k].pdf(data[i]);
            total += resp[i][k];
        }
        // Step 2: 归一化
        if (total > 1e-300) {
            for (int k = 0; k < K; k++) {
                resp[i][k] /= total;
            }
        } else {
            // 数值安全：如果所有分量的 PDF 都极小，退化为均匀分配
            for (int k = 0; k < K; k++) res[i][k] = 1.0 / K;
        }
    }
    return resp;
}
```

**为什么用 `total > 1e-300` 而不是 `total > 0`？** 当数据点离所有分量都很远时，所有 PDF 值都极小（比如 \(10^{-200}\)），但 double 能精确表示。只有当真接近 0（如 \(10^{-310}\)，即 subnormal 以下）才会出问题。这个阈值既保证数值安全，又避免了无谓的退化。

**为什么退化为均匀分配而不是报错？** 如果某个数据点是真正的"离群点"，报错会终止整个训练（丢失其他合理数据的信息）。均匀分配是最无偏的猜测——不给任何分量偏见。

### 4.2 多变量高斯 PDF

```cpp
double pdf(Vec2 x) const {
    Vec2 d = x - mean;  // 中心化
    double exponent = -0.5 * dot(d, mul(cov_inv, d));  
    return std::exp(exponent) / (2.0 * M_PI * std::sqrt(det_cov));
}
```

**关键细节**：
- `dot(d, mul(cov_inv, d))` 计算的是**马氏距离的平方** \((x-\mu)^T\Sigma^{-1}(x-\mu)\)。如果协方差矩阵很大（聚类分散），马氏距离就小——这自然地让宽聚类的 PDF 值在各处都更高，补偿了归一化常数。
- 归一化常数 \(1/(2\pi\sqrt{|\Sigma|})\) 确保 PDF 在整个空间的积分为 1。如果忘记除以这个值，PDF 就不再是有效的概率密度。

### 4.3 M-step：协方差更新（含正则化）

```cpp
void m_step(/* ... */) {
    // ... 均值更新 ...
    
    for (int k = 0; k < K; k++) {
        if (Nk[k] < 1e-10) {
            // 处理"死亡分量"：如果没几个点属于该分量，重新初始化
            components[k].cov = mat2(1.0, 0, 0, 1.0);
            components[k].weight = 0.0;
        } else {
            Mat2 new_cov = {0, 0, 0, 0};
            // 计算责任度加权的外积 (x_i - μ_k)(x_i - μ_k)^T
            for (int i = 0; i < N; i++) {
                Vec2 diff = data[i] - components[k].mean;
                new_cov = new_cov + outer(diff) * resp[i][k];
            }
            new_cov = new_cov * (1.0 / Nk[k]);
            // 正则化：对角线加 ε，防止奇异
            new_cov.a += 1e-6;
            new_cov.d += 1e-6;
            components[k].cov = new_cov;
            components[k].weight = Nk[k] / total_resp;
        }
        components[k].update_cache();
    }
}
```

**为什么协方差更新用外积？** 协方差矩阵定义为 \(E[(X-\mu)(X-\mu)^T]\)。对于一维数据，这就是方差；对于高维数据，外积 \(\text{outer}(diff) = diff \cdot diff^T\) 产生了矩阵的每个元素 \((diff_i \cdot diff_j)\)，乘以责任度做加权平均。

**"死亡分量"问题**：如果初始化不当，某个分量可能"抢"不到数据（所有责任度都接近 0），它的 \(N_k\) 就会趋近 0。此时更新协方差会出现除零错误。解决方案是检测到 \(N_k < 10^{-10}\) 时，将该分量重置为"空"（权重 0），让其他分量接管。如果确实需要更多分量，BIC 会在模型选择时反映出来。

**对角线正则化**：2×2 协方差矩阵有两个自由度在对角线上。如果某个方向上完全没有变化（比如聚类恰好是一条直线），协方差矩阵就退化（行列式为 0），无法求逆。给对角线加一个小正数 \(\varepsilon = 10^{-6}\) 相当于加了微小的"先验知识"（假设每个方向至少有一点点变化），这在贝叶斯视角下是一种共轭先验。

### 4.4 EM 训练主循环

```cpp
void fit(const std::vector<Vec2>& data, int max_iter = 200, double tol = 1e-5) {
    double prev_ll = -std::numeric_limits<double>::infinity();
    
    for (int iter = 0; iter < max_iter; iter++) {
        auto resp = e_step(data);         // ① 计算责任度
        m_step(data, resp);               // ② 更新参数
        
        double ll = compute_log_likelihood(data);  // ③ 计算对数似然
        
        if (std::abs(ll - prev_ll) < tol) {
            log_likelihood = ll;
            bic = compute_bic(data);
            return;  // 收敛！
        }
        prev_ll = ll;
    }
    // 达到最大迭代次数
    log_likelihood = prev_ll;
    bic = compute_bic(data);
}
```

**收敛检测 vs K-Means**：K-Means 收敛通常需要几千次迭代（每次只移动中心一小步）。GMM 的 EM 算法通常在 10-50 次迭代内就收敛到高精度，因为 E-step 的软指派让 M-step 的梯度信息更丰富。

### 4.5 BIC 计算与模型选择

```cpp
double compute_bic(const std::vector<Vec2>& data) {
    double ll = compute_log_likelihood(data);
    int n_params = 6 * K - 1;  // 2D: K*(1+2+3)-1
    int N = data.size();
    return -2.0 * ll + n_params * std::log(static_cast<double>(N));
}
```

在 main 中：
```cpp
for (int K : {2, 3, 4, 5}) {
    GMM gmm(K);
    // K-Means++ 风格初始化（第一个中心随机，后续选择最远点）
    // ...
    gmm.fit(data);
    if (gmm.bic < best_bic) {
        best_bic = gmm.bic;
        best_K = K;
    }
}
```

**为什么对数惩罚项是 \(\log N\)？** 这个设计来自信息论：模型复杂度惩罚应当随着数据量的对数增长。直觉上，如果你有 10 个点，多用一个参数应该受到较大的惩罚（因为数据太少，容易过拟合）；但如果你有 10^6 个点，多用一个参数几乎不影响泛化能力。\(\log N\) 正好体现了这种"边际递减"的惩罚效应。

### 4.6 数据生成：给定真实参数的合成数据

为了验证算法的正确性，我们需要从"已知真值"的参数中生成数据：

```cpp
// 三个分离良好的聚类
true_means = {{1.0, 2.0}, {8.0, 8.0}, {5.0, 1.0}};
true_covs  = {
    {0.8, 0.3, 0.3, 0.8},   // 倾斜椭圆
    {0.7,-0.2,-0.2, 0.9},   // 另一个方向的倾斜
    {0.6, 0.0, 0.0, 0.5}    // 轴对齐的椭圆
};
true_weights = {0.33, 0.33, 0.34};
```

从协方差矩阵生成样本的数学原理是**协方差矩阵的 Cholesky 分解**：如果 \(L\) 满足 \(\Sigma = LL^T\)，且 \(z \sim \mathcal{N}(0, I)\)，则 \(x = \mu + Lz \sim \mathcal{N}(\mu, \Sigma)\)。

在我们的 2D 实现中，Cholesky 分解等价于**特征分解**：找到特征值 \(\lambda_1, \lambda_2\) 和特征向量 \(v_1, v_2\)，然后用标准正态分布缩放：
\[
x = \mu + \sqrt{\lambda_1} \cdot z_1 \cdot v_1 + \sqrt{\lambda_2} \cdot z_2 \cdot v_2
\]
其中 \(z_1, z_2 \sim \mathcal{N}(0, 1)\) 独立同分布。这保证生成的数据精确服从 \(\mathcal{N}(\mu, \Sigma)\)。

## ⑤ 踩坑实录

### 坑 1：协方差矩阵变成奇异（初次运行崩溃）

**症状**：在 K=4 和 K=5 的训练中，某个分量的协方差矩阵行列式变成 0，导致 `inv()` 函数除零崩溃。

**错误假设**：我以为 M-step 总能得到有效的协方差矩阵——毕竟"经验协方差"公式本身是良定义的。

**真实原因**：某个分量"死亡"了——所有数据点的责任度都接近于 0，导致 \(N_k \approx 0\)，协方差更新时分母趋近于 0，数值不稳定导致退化。

**修复方式**：
1. 在 M-step 中检测 \(N_k < 10^{-10}\)，如果触发就重置该分量为单位矩阵
2. 对协方差对角线加 \(10^{-6}\) 的正则化项（一个非常小的值，不影响正常聚类但能在极端情况下保底）
3. 提高收敛容差从 \(10^{-6}\) 到 \(10^{-5}\)，让 EM 在达到合理精度后就停止，而不是无限迭代

### 坑 2：BIC 选择了 K=4 而不是真正的 K=3

**症状**：第一次运行中，BIC 在 K=4 时取得最小值（BIC4=4831.4 < BIC3=4877.5），而数据的真实聚类数是 3。

**错误假设**：我认为 BIC 的惩罚项总能阻止"多用一个分量"。

**真实原因**：初始的合成数据三个聚类靠得太近（中心距离约 3-5 个标准差），有一部分点处于聚类的模糊边界上。在这些边界区域，K=4 比 K=3 多获得了约 10 个单位的对数似然改善，而 6 个额外参数（23 vs 17）乘以 \(\log(600) \approx 6.4\) 约增加 38 的惩罚——但似然改善（约 52 个单位）超过了惩罚。

**修复方式**：将三个聚类的中心距离拉大（中心从 (2,3),(8,7),(5,1.5) 改为 (1,2),(8,8),(5,1)），同时缩小协方差矩阵的幅值（标准差从 ~1.0 缩小到 ~0.8）。这样三个聚类的 95% 置信椭圆几乎完全分离，BIC 正确选择 K=3。

**这个坑的教训**：BIC 不是万能的——它假设数据可以从模型中生成，且样本量足够大。当聚类高度重叠时，BIC 可能在"用更多分量去适配重叠"和"用更少分量接受噪声"之间做出权衡。算法验证时，确保合成数据的聚类是可分离的，才能验证算法本身是否正确。

### 坑 3：初始化导致收敛到局部最优

**症状**：某些运行中，两个分量"抢"到了同一个真实聚类，而另一个真实聚类没有被任何分量覆盖（两个分量挤在一起）。

**错误假设**：随机初始化就够了。

**真实原因**：EM 算法只能保证收敛到**局部最优**，不是**全局最优**。如果初始化的中心恰好都在同一个真实聚类附近，EM 会把它们分成两个"亚聚类"，而远处的真实聚类完全被忽略。

**修复方式**：实现了 K-Means++ 风格的初始化——第一个中心随机选择，后续每个中心选择"离已有中心最远的点"。这在概率上更可能分散初始中心到不同聚类区域，显著降低了局部最优的概率。

### 坑 4：对数似然收敛判定太严格

**症状**：K=3 的训练跑了 100 次迭代（老版本 max_iter=100），最后几次迭代的对数似然变化在 \(10^{-6}\) 到 \(10^{-5}\) 之间，差一点就达到 \(10^{-6}\) 的阈值。

**错误假设**：我把收敛容差设得太严格（tol=1e-6）。

**真实原因**：double 精度下，对数似然的变化在 \(10^{-5}\) 级别已经是"数值收敛"了——再往下走只是浮点噪声。对 600 个数据点，每个点的 PDF 计算包含 exp、除法等操作，累积的数值误差约在 \(10^{-14}\)，乘以 600 后到总对数似然的误差约 \(10^{-11}\)，但在 \(\Delta\log L\) 中会被放大。

**修复方式**：将 tol 从 \(10^{-6}\) 放宽到 \(10^{-5}\)，并将 max_iter 从 100 提高到 200。结果显示 K=3 在 15 次迭代后就收敛了——放宽容差反而让收敛检测更合理。

## ⑦ 总结与延伸

### GMM 相比 K-Means 的核心优势

1. **软聚类**：每个点属于各类的概率，而非硬指派
2. **表达任意形状的聚类**：通过协方差矩阵建模椭球形（而非球形）
3. **概率框架**：可以计算新数据点的生成概率（用于异常检测）
4. **BIC 自动选择 K**：不必预先知道聚类数

### 局限性

1. **EM 只能保证局部最优**：初始化好坏直接影响结果，随机重启 + BIC 选最佳是一种缓解策略
2. **假设数据服从混合高斯分布**：现实中很多数据不满足这个假设（如环状聚类、流形结构），此时需要用 DBSCAN 或谱聚类等非参数方法
3. **协方差矩阵的参数数量是 \(O(D^2)\)**：在高维数据（如 D=1000）中，每个分量的协方差矩阵有 50 万参数——这在计算和存储上都不现实。解决方案是使用对角的 \(\Sigma_k\)（等价于各维度独立）或共享协方差矩阵
4. **对离群点敏感**：EM 的高斯假设意味着离群点（远离所有均值）也能以极小概率被"接纳"——这虽然不崩溃，但会扭曲协方差估计

### 与系列其他文章的关联

- **K-Means (06-21)**：GMM 的"硬聚类"版本，K-Means 可以看作是 GMM 在等方差球形假设下的极限——当所有分量的协方差矩阵都趋近于 \(\sigma^2 I\) 且 \(\sigma \to 0\) 时，EM 退化为 K-Means 的 Lloyd 迭代
- **DBSCAN (06-22)**：Density-based 聚类，处理非凸形状。GMM 假设椭球形，DBSCAN 不假设任何形状，但不提供概率解释
- **谱聚类**（未来可选）：用图拉普拉斯的特征向量做聚类，天然处理流形数据，可与 GMM 的协方差建模结合形成"GMM on eigenmap"

### 可继续优化的方向

1. **对角协方差 GMM**：在高维数据中强制 \(\Sigma_k\) 为对角矩阵（\(O(D)\) 参数），实践中效果往往不差
2. **变分贝叶斯 GMM (VB-GMM)**：给参数加先验分布（Wishart 先验用于协方差、Dirichlet 先验用于权重），自动消除冗余分量
3. **K-Means++ 重启**：多次随机初始化 + BIC 评分取最优，降低局部最优风险
4. **在线 EM**：对大数据流，每来一个 batch 做一次 E/M 步，不存储历史数据
